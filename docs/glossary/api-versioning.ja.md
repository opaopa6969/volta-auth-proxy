# API バージョニング

[English version](api-versioning.md)

---

## これは何？

API バージョニングとは、既存のクライアントを壊さずに API を進化させる戦略です。エンドポイントの動作を変更する必要がある場合、既存のものを修正するのではなく、新しいバージョンをリリースします。既存クライアントは v1 を使い続け、新しいクライアントは v2 を使えます。

レストランのメニューと同じです。料理のレシピを変えるとき、差し替える（常連がお気に入りだった体験を壊す）か、元のメニューと並行して「新バージョン」を追加するかを選べます。

---

## なぜ重要？

API には**利用者**がいます -- モバイルアプリ、フロントエンドコード、パートナー連携。これらの利用者があなたの API に対して構築された後、API を変更するとそれらが壊れます。バージョニングがないと：

- 何百万台のスマホにデプロイされたモバイルアプリは即座に更新できない
- パートナー連携が深夜3時のデプロイで動かなくなる
- ブラウザにキャッシュされたフロントエンドコードがバックエンド更新後に失敗する

バージョニングにより、利用者に移行の時間を与えながら API を進化させることができます。

## 簡単な例

### 一般的なバージョニング戦略

| 戦略 | 例 | メリット | デメリット |
|------|-----|---------|----------|
| **URL パス** | `/api/v1/users` | 明白、ルーティングが容易 | エンドポイントが重複 |
| **ヘッダー** | `Accept: application/vnd.myapi.v2+json` | クリーンな URL | 発見しにくい、ブラウザでテストしにくい |
| **クエリパラメータ** | `/api/users?version=2` | シンプル | 付け忘れやすい |

### 何が破壊的変更？

| 変更 | 破壊的？ |
|------|---------|
| レスポンスに新しいフィールドを追加 | いいえ |
| 新しいオプションパラメータを追加 | いいえ |
| レスポンスからフィールドを削除 | **はい** |
| フィールド名を変更 | **はい** |
| フィールドの型を変更（文字列 -> 数値） | **はい** |
| ステータスコードの意味を変更 | **はい** |
| 新しいエンドポイントを追加 | いいえ |

## volta-auth-proxy では

volta はすべての API エンドポイントを `/api/v1/` 以下に配置する **URL パスバージョニング**を使用しています：

```
GET  /api/v1/users/me
GET  /api/v1/tenants/{tenantId}/members
POST /api/v1/tenants/{tenantId}/invitations
GET  /api/v1/admin/tenants
POST /api/v1/admin/keys/rotate
```

### 追加のみの JWT クレームポリシー

volta は JWT クレームのバージョニングに重要なアプローチを取っています：**追加のみの変更**。すべての JWT の `volta_v` クレームがスキーマバージョンを示します（現在 `1`）：

```json
{
  "volta_v": 1,
  "volta_tid": "...",
  "volta_roles": ["ADMIN"]
}
```

ルール：
- 新しいクレームの**追加**（例：`volta_permissions`）はどのバージョンでも安全
- クレームの**削除**や**リネーム**は `volta_v` を `2` に上げる必要がある
- 下流サービスは `volta_v` を確認し、移行中は両方のフォーマットを処理すべき

つまり、`volta_v: 1` の JWT 向けに構築されたサービスは、新しいクレームが追加されても動き続けます。バージョン番号が変わった時だけ更新が必要です。

### 破壊的変更ポリシー

volta は以下の原則に従います：
1. **v1 は安定を維持。** バージョン内で破壊的変更なし。
2. **新バージョンは追加的。** v2 は機能を追加するが、v1 エンドポイントを即座に削除しない。
3. **削除前に非推奨化。** v1 を最終的に廃止する場合は、十分前もって告知。
4. **JWT クレームは独立してバージョニング。** `volta_v` で管理し、API パスバージョンとは別。
