# 同意画面（Consent Screen）

[English / 英語](consent-screen.md)

---

## これは何？

同意画面は、ID プロバイダ（Google など）がユーザーに表示するページで、アプリがどのデータにアクセスしたいか承認を求めます。「volta があなたのメールアドレスと基本的なプロフィール情報の閲覧を希望しています -- 許可しますか？ 拒否しますか？」のような内容です。OIDC フローが続行される前に、ユーザーは明示的に許可を与える必要があります。データが共有される前に「いいえ」と言えるユーザーの機会です。

---

## なぜ重要？

同意画面が存在するのは、根本的な原則があるからです：ユーザーは自分のデータを誰が見るか制御できるべきである。同意画面がなければ、Google の client ID を知っているアプリなら何でも、無音でメールやプロフィールにアクセスできてしまいます。同意画面は人間によるチェックポイントを作ります -- 攻撃者がどれほど巧妙にフィッシングリンクを作成しても、ユーザーは何が要求されているか確認でき、拒否できます。

同意画面は説明責任も生みます。ユーザーは自分のデータにアクセスしているアプリを正確に把握でき、プロバイダの設定からアクセスを取り消すことができます。

---

## 簡単な例

volta がユーザーを Google にリダイレクトすると、Google が表示：

```
+------------------------------------------+
|  アカウントを選択                          |
|                                          |
|  alice@gmail.com                         |
|  bob@company.com                         |
|                                          |
+------------------------------------------+
         |  (ユーザーがアカウントを選択)
         v
+------------------------------------------+
|  volta が以下を希望しています：              |
|                                          |
|  - メールアドレスの閲覧                    |
|  - 個人情報（名前、写真）の閲覧             |
|                                          |
|  [許可]              [拒否]               |
+------------------------------------------+
```

volta は `prompt=select_account` を指定し、アカウントが1つしかなくても常にアカウント選択画面を表示します。複数アカウントの場面で、意図しないアカウントへの無音ログインを防ぎます。

---

## volta-auth-proxy での使い方

volta は2つのレベルで同意に関わります：

**Google の同意画面**（OIDC レベル）：

volta は最小限のスコープ（`openid email profile`）を要求し、Google の同意画面をシンプルに保ちます。`prompt=select_account` パラメータでアカウント選択画面を強制表示：

```java
params.put("prompt", "select_account");
```

これは意図的な UX の選択です。これがないと、既に Google にサインイン済みのユーザーがプロンプトなしに無音ログインされ、誤ったアカウントでログインしてしまう可能性があります。

**volta 独自の招待同意**（アプリケーションレベル）：

volta にはワークスペース招待の独自の同意概念があります。ユーザーがテナントに招待されると、招待承認ページが表示されます。これは Google の同意画面ではなく、volta 独自の UI で「ワークスペース X に招待されました。承認しますか？」と尋ねます。招待コードが OIDC フローに保存され、承認ページは volta の jte テンプレートでレンダリングされます。

2つの同意レイヤーは異なる目的を持ちます：Google の同意はユーザーの Google データを保護。volta の招待同意はワークスペースを不正な参加から保護します。
