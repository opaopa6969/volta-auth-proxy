## これは何？

**データベースは、コンピュータ用の超整理されたファイリングキャビネットです。情報を見つけやすく、更新しやすく、管理しやすい形で保存します。**

知らなくても大丈夫です！データベースはいたるところにあり、銀行の残高からSNSの投稿まで、あらゆるものを静かに保存しています。

## 身近なたとえ

学校のクラス名簿を思い浮かべてください：

```
  ┌─────────────────────────────────────────────────────┐
  │  3年A組 - 生徒名簿                                    │
  │                                                     │
  │  生徒番号 │ 名前          │ 年齢 │ 好きな色          │
  │  ────────┼──────────────┼──────┼─────────────────  │
  │  001     │ 田中ゆき       │ 9    │ 青                │
  │  002     │ 鈴木ひろ       │ 9    │ 赤                │
  │  003     │ 渡辺めい       │ 10   │ 緑                │
  │  004     │ 佐藤けん       │ 9    │ 青                │
  └─────────────────────────────────────────────────────┘
```

この名簿は、基本的にデータベースの**テーブル**です！用語を覚えましょう：

| データベースの用語 | クラス名簿の対応 |
|--------------|------------------------|
| **テーブル** | 名簿全体 |
| **行（ロウ）** | 一人の生徒の情報（横一列） |
| **列（カラム）** | 情報の種類（名前、年齢など） |
| **レコード** | 行と同じ意味 -- 一件分のデータ |

## なぜ表計算ソフトじゃダメなのか？

表計算ソフト（Excelなど）は少量のデータには最適です。でも想像してみてください：

- 50校にまたがる10,000人の生徒
- 何百人もの先生が同時に編集
- 「青が好きな9歳の生徒全員」を一瞬で見つける必要がある
- 停電しても絶対にデータを失ってはいけない

ここでデータベースが力を発揮します。速度、安全性、そして多くのユーザーの同時利用のために作られています。

## データベースは何ができる？

とても賢いファイリング係だと思ってください：

```
  あなた：「3年A組の9歳の生徒を全員見つけて」
  DB：    はい：ゆき、ひろ、けんです！

  あなた：「めいの年齢を10歳に変えて」
  DB：    完了！更新しました。

  あなた：「新しい生徒を追加して：さくら、9歳、紫が好き」
  DB：    追加しました！生徒番号005です。

  あなた：「青が好きな生徒は何人？」
  DB：    2人です！
```

## PostgreSQL（Postgres）とは？

PostgreSQL（よく「Postgres」と呼ばれます）は、世界で最も人気のあるデータベースの一つです：

- **無料** -- オープンソースです（[open-source](open-source.ja.md)を参照）
- **信頼性が高い** -- 銀行や政府が信頼して使っている
- **高性能** -- 何百万件ものレコードを扱える

データベース界の「トヨタ」のようなものです。信頼性が高く、しっかり作られていて、広く信頼されています。

## volta-auth-proxyでは

**volta-auth-proxyでは：** PostgreSQLがすべての重要なデータを保存しています。ユーザーアカウント、ログインセッション、テナント設定、セキュリティ設定など。voltaがユーザーを安全に守るために必要なすべてを覚えている「ファイリングキャビネット」です。
