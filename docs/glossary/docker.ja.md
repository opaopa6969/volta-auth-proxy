## これは何？

**Dockerは、プログラムとそれが動くために必要なものすべてを1つの箱にまとめる方法です。どの船に載せても同じように使える輸送コンテナのようなものです。**

知らなくても大丈夫です！Dockerは主に開発者が裏側で使うツールで、ソフトウェアのセットアップと実行を簡単にします。

---

## 身近なたとえ

おばあちゃんのケーキのレシピを外国の友達にシェアしたいとします：

**Dockerなし（昔のやり方）：**
```
  「レシピはこれ！あ、でもこの特定のブランドの小麦粉が必要で、
  あなたのオーブンは私のと違うし、特別なミキサーを買わないと
  いけないし、あなたの住んでる場所の標高で焼き時間が変わるし...」

  結果：3時間トラブルシューティングしても、ケーキの味がおかしい。
```

**Dockerあり：**
```
  「この魔法の箱をどうぞ。レシピ、材料、正確なオーブン、
  ミキサー、正しい標高まで全部入ってます。スタートを
  押すだけ。」

  結果：いつでも、どこでも、完璧なケーキ。
```

Dockerはこの魔法の箱です。プログラムに必要なもの全て -- 正しいバージョンのソフトウェア、正しい設定、すべて -- をパッケージにするので、どのコンピュータでも同じように動きます。

## 輸送コンテナのたとえ

```
  輸送コンテナ登場前：               輸送コンテナ登場後：

  ┌─────────────────────┐            ┌─────────────────────┐
  │ 手作業で積み込み      │            │ ┌─────┐ ┌─────┐    │
  │ 荷物の種類ごとに      │            │ │箱   │ │箱   │    │
  │ やり方が違う          │            │ │ 1  │ │ 2  │    │
  │ 輸送中に壊れる        │            │ └─────┘ └─────┘    │
  │ 時間がかかる          │            │ 標準サイズ           │
  └─────────────────────┘            │ どの船でもOK         │
                                     │ どのトラックでもOK    │
                                     └─────────────────────┘
```

実際の輸送コンテナが発明される前は、船への積み込みは混乱そのものでした。荷物はそれぞれ違いました。輸送コンテナがすべてを標準化しました：同じサイズ、同じ積み方、どの船・トラック・列車でも使えます。

Dockerはソフトウェアに対して同じことをします。

---

## 開発者がDockerを好きな理由

ソフトウェア界で最も有名な問題：

```
  開発者A：「私のコンピュータでは動きますよ！」
  開発者B：「私のでは動かないんですけど。」
```

Dockerはこの問題をなくします。Dockerコンテナの中で動けば、どこでも動きます。

## Dockerの用語（かんたん版）

| 用語 | 意味 |
|------|---------|
| **イメージ** | レシピ＋材料（設計図） |
| **コンテナ** | イメージの実行中のコピー（実際のケーキ） |
| **Dockerfile** | イメージを作る手順書（レシピカード） |
| **docker-compose** | 複数のコンテナをまとめて実行する方法（献立表） |

## volta-auth-proxyでは

**volta-auth-proxyでは：** Dockerは開発環境全体の実行に使われています。voltaサーバー、PostgreSQLデータベース、その他のサービスがすべてDockerコンテナで動くので、どのコンピュータを使っていても1つのコマンドですべてを起動できます。
