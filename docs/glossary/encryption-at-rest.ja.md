# 保存時暗号化（Encryption at Rest）

[English / 英語](encryption-at-rest.md)

---

## これは何？

保存時暗号化（Encryption at Rest）とは、データが保存されている状態（ディスク上、データベース内、バックアップテープ上）で保護することです。通信中のデータ（ネットワーク上を移動するデータ）の保護とは対になる概念です。誰かがストレージへの物理アクセスを得たり、データベースのダンプが漏洩したりしても、復号鍵がなければデータは読めません。家に鍵をかけているのに加えて、貴重品を金庫にも入れるようなものです。

---

## なぜ重要？

データベースは侵害されます。バックアップは盗まれます。ディスクは適切にワイプされずに廃棄されます。暗号署名鍵のような機密データが平文で保存されていると、ストレージにアクセスした人はシステムを偽装するために必要なものすべてを手に入れます。保存時暗号化は第二の防衛層を追加します：侵害後も、攻撃者が得るのは暗号文であり、使える鍵ではありません。

volta のような認証ゲートウェイにとって、最も機密性の高いデータは JWT に署名する RSA 秘密鍵です。攻撃者がこの鍵を入手すると、任意のユーザーのトークンを偽造できます。保存時暗号化により、データベース侵害だけでは不十分になります。

---

## 簡単な例

保存時暗号化なし：
```
データベーステーブル: signing_keys
+--------+---------------------------+
| kid    | private_key_pem           |
+--------+---------------------------+
| key-1  | MIIEvgIBADANBgkqhki...   |  <-- 平文、すぐに使える
+--------+---------------------------+
```

保存時暗号化あり（AES-256-GCM）：
```
データベーステーブル: signing_keys
+--------+------------------------------------------+
| kid    | private_key_pem                          |
+--------+------------------------------------------+
| key-1  | v1:BASE64_IV:BASE64_暗号化ペイロード        |  <-- 鍵がなければ無意味
+--------+------------------------------------------+
```

暗号化されたバージョンは、データベースの外にある環境変数 `JWT_KEY_ENCRYPTION_SECRET` がなければ無意味です。

---

## volta-auth-proxy での使い方

volta は RSA 署名鍵を PostgreSQL に保存する前に `KeyCipher` で暗号化します。AES-256-GCM を実装しています：

```java
public String encrypt(String plain) {
    byte[] iv = new byte[12];            // ランダムな 96 ビット IV
    RANDOM.nextBytes(iv);
    Cipher cipher = Cipher.getInstance("AES/GCM/NoPadding");
    cipher.init(Cipher.ENCRYPT_MODE, keySpec, new GCMParameterSpec(128, iv));
    byte[] encrypted = cipher.doFinal(plain.getBytes(StandardCharsets.UTF_8));
    return "v1:" + b64(iv) + ":" + b64(encrypted);
}
```

重要な詳細：

- **アルゴリズム**: AES-256-GCM（認証付き暗号化 -- 機密性と完全性の両方を保証）
- **鍵導出**: 環境変数 `JWT_KEY_ENCRYPTION_SECRET` を SHA-256 でハッシュし、256ビットの AES 鍵を生成
- **IV（初期化ベクトル）**: 暗号化操作ごとにランダムな12バイトの IV を生成。同じ平文でも毎回異なる暗号文になる
- **フォーマット**: `v1:BASE64_IV:BASE64_暗号文` -- `v1:` プレフィックスで将来のフォーマット変更に対応
- **後方互換性**: 保存された値が `v1:` で始まらない場合、`decrypt()` はそのまま返す（暗号化なし保存からの移行用）

暗号化秘密（`JWT_KEY_ENCRYPTION_SECRET`）は環境変数で設定され、データベースには保存されません。つまり攻撃者は署名鍵を復号するために、データベースダンプとアプリケーション環境の両方が必要です。
