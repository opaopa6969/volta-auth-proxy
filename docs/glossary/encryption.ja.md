# 暗号化（Encryption）

[English version](encryption.md)

---

## これは何？

**暗号化は、読めるテキストを正しい鍵を持つ人だけが元に戻せる意味不明な文字列に変えることです。親友だけが知っている秘密の暗号で書くようなものです。**

知らなくても大丈夫です！暗号化は毎日あなたを守っています。オンラインショッピング、メッセージの送信、ウェブサイトへのログイン。裏側で静かに働いています。

---

## 身近なたとえ

鍵付きの日記帳を思い浮かべてください：

```
  ┌────────────────────────────────────────────────────┐
  │                                                    │
  │  あなたの日記（原文）：   「ピザが好き」              │
  │                                                    │
  │         │                                          │
  │         ▼  ロック（鍵で暗号化）                      │
  │                                                    │
  │  ロックされた日記：       「xK9$mQ2!pL」             │
  │                                                    │
  │  誰かが見つけた：        「xK9$mQ2!pLって           │
  │                          何？？わからない！」        │
  │                                                    │
  │         │                                          │
  │         ▼  アンロック（鍵で復号）                    │
  │                                                    │
  │  友達（鍵を持っている）：「ピザが好き」               │
  │                         「なるほど！」              │
  │                                                    │
  └────────────────────────────────────────────────────┘
```

- **暗号化** = 日記にロックをかける（読めるテキストを意味不明にする）
- **復号** = 日記のロックを外す（意味不明なテキストを読めるものに戻す）
- **鍵** = ロックとアンロックに使うもの

鍵がなければ、スクランブルされたテキストは無価値です。たとえ誰かが盗んでも。

## 簡単な例

とても基本的な暗号化を想像してください：すべての文字を3つずらす。

```
  原文：     HELLO
  暗号化後： KHOOR    (H→K, E→H, L→O, L→O, O→R)
```

ルール（「3つずらす」）を知っていれば復号できます。知らなければ「KHOOR」は何の意味もありません。

実際の暗号化はこれよりはるかに複雑です（コンピュータは解読に何百万年もかかる数学的パズルを使います）。でもアイデアは同じです。

## 暗号化はどこで使われている？

暗号化はどこにでもあります：

```
  🔒 オンラインショッピング  - クレジットカード番号が暗号化される
  🔒 メッセージアプリ       - メッセージがエンドツーエンドで暗号化される
  🔒 ウェブサイトのログイン   - パスワードが送信中に暗号化される
  🔒 銀行アプリ            - すべての金融データが暗号化される
  🔒 Wi-Fiパスワード       - 自宅のWi-Fiが通信を暗号化する
```

ブラウザのアドレスバーにある小さな南京錠のアイコン、あれは「あなたとウェブサイトの間の通信が暗号化されている」という意味です。

---

## なぜ暗号化は大切なのか？

暗号化なし：

```
  あなた：     「パスワードは'sushi123'」
  インターネット：サーバーに届くまで20台のコンピュータを通過
  ハッカー：   「読めた！パスワードは'sushi123'だ！」
```

暗号化あり：

```
  あなた：     「パスワードは'sushi123'」→ 暗号化して「aX$9kL!mP2」
  インターネット：20台のコンピュータを通過
  ハッカー：   「見えるのは... aX$9kL!mP2。意味が分からない。」
  サーバー：   「aX$9kL!mP2」を復号 → 「sushi123」→「ログイン成功！」
```

## volta-auth-proxyでは

**volta-auth-proxyでは：** 暗号化はログイントークンを検証する署名鍵の保護に使われています。たとえ誰かがデータベースにアクセスできたとしても、復号鍵がなければ暗号化された鍵は使い物にならず、認証システムの安全が保たれます。
