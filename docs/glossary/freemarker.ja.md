# FreeMarker

[English version](freemarker.md)

---

## これは何？

**FreeMarkerは、テンプレートの空欄を埋めるツールです。Microsoft Wordの「差し込み印刷」と同じようなものです。**

知らなくても大丈夫です！FreeMarkerはニッチなツールで、ほとんどの人は聞いたことがありません。身近な例から始めましょう。

---

## 身近なたとえ

学校で働いていて、200人の保護者に同じ手紙を送る必要があるとします。ただし、それぞれの手紙にはお子さんの名前とクラスが必要です：

```
______様

お子様の______さん（______組）は、今学期とてもがんばっています！

学校より
```

200通を手書きはしませんよね。代わりに**差し込み印刷**を使います：空欄のあるテンプレートを1つ作って、名前とクラスのリストからコンピュータが自動で埋めてくれます。

**FreeMarkerはまさにこれを、手紙ではなくウェブページに対して行います。**

---

## どんな見た目？

ログインページ用のFreeMarkerテンプレートの簡単な例：

```
おかえりなさい、${username}さん！
新しいメッセージが${messageCount}件あります。
```

ユーザー名が「田中」でメッセージが3件なら、FreeMarkerはこう出力します：

```
おかえりなさい、田中さん！
新しいメッセージが3件あります。
```

`${...}`の部分が、自動的に埋められる「空欄」です。

---

## なぜKeycloakはFreeMarkerを使うのか？

Keycloak（volta-auth-proxyが置き換えるログインシステム）は、ログインページ、エラーページ、メールテンプレートの作成にFreeMarkerを使っています。テンプレートを一度書けば、Keycloakがユーザー名やエラーメッセージなどを自動で埋めてくれるという考え方です。

## なぜFreeMarkerは大変なのか？

先ほどの学校の手紙テンプレートを想像してください。でも今度は：

- テンプレートが500行もある
- 空欄に`${realm.displayNameHtml!''}`のような分かりにくい名前がついている
- どのページでどの空欄が使えるかに47個の隠れたルールがある
- 小さなミスでページ全体が意味不明なエラーで壊れる
- 送る前に手紙がどう見えるか確認する良い方法がない

```
  Keycloakの実際のFreeMarkerテンプレート：

  <#if realm.password && social.providers??>     <-- これ何！？
    <div id="${properties.kcFormSocialAccountSectionClass!}">
      <#list social.providers as p>
        <a href="${p.loginUrl}">${p.displayName!}</a>
      </#list>
    </div>
  </#if>
```

プロの開発者にとっても面倒です。ログインページのボタンの色を変えたいだけの人にとっては、悪夢です。

---

## voltaは代わりに何を使っている？

volta-auth-proxyは**jte**（Java Template Engine）を使っています。よりシンプルで高速、そしてページがユーザーに表示される前にミスを検出してくれます。200通の手紙を送る前に「お子さんの名前の欄のスペルが間違っていますよ！」と教えてくれる、賢い差し込み印刷のようなものです。

---

## volta-auth-proxyでは

**volta-auth-proxyでは：** FreeMarkerは一切使っていません。voltaはKeycloakのFreeMarkerテンプレートをjteテンプレートに置き換え、ログインページのカスタマイズをずっと簡単に、壊れにくくしました。
