## これは何？

**Gitはファイルのタイムマシンです。いつでも作業を保存でき、いつでも以前の保存に戻れます。**

知らなくても大丈夫です！Gitは開発者がコードの変更を追跡するために使うツールです。究極の「元に戻す」ボタンだと思ってください。

---

## 身近なたとえ

小説を書いていると想像してください：

```
  月曜日：第1章を書いた。              [セーブポイント1]
  火曜日：第2章を書いた。              [セーブポイント2]
  水曜日：第2章を書き直して、           [セーブポイント3]
          ずっと良くなった。
  木曜日：うっかり全部消して、           大変！
          ひどいものを書いてしまった。

  Gitがあれば：「水曜日のバージョンに戻して。」
               パッ！すべて元通り。
```

Gitがなければ、今までの変更をすべて覚えているか、`小説_v1.doc`、`小説_v2.doc`、`小説_v2_最終版.doc`、`小説_v2_最終版_本当の最終版.doc`のように保存するしかありません...

誰でもやったことがありますよね！

## 大事な概念（専門用語なし！）

### セーブポイント = 「コミット」

**コミット**は、ゲームで「セーブ」を押すようなものです。その時点ですべてがどうなっているかを記録します。何が変わったか短いメモを付けます：

```
  セーブポイント1：「ログインページを追加した」
  セーブポイント2：「パスワードがチェックされないバグを直した」
  セーブポイント3：「エラーメッセージを分かりやすくした」
```

### 自分の作業を共有する = 「プッシュ」

**プッシュ**すると、セーブポイントをGitHubのような共有場所に送り、チームメイトが見れるようになります。小説の章を共有フォルダに入れるようなものです。

```
  自分のPC  ----プッシュ---->  共有場所（GitHub）
                                （チームメイトが見れる）
```

### 他の人の作業をもらう = 「プル」

**プル**すると、チームメイトの最新のセーブポイントを自分のコンピュータにダウンロードします。

```
  自分のPC  <----プル----  共有場所（GitHub）
                            （チームメイトの変更をもらう）
```

## なぜ開発者はGitを使うのか

```
  Gitなし：                          Gitあり：

  レポート_最終版.doc                 ファイルは1つで、
  レポート_最終版_v2.doc              すべての変更の完全な履歴あり。
  レポート_最終版_v2_修正済.doc        いつの時点にも戻れる。
  レポート_消すな.doc                 誰がいつ何を変えたか分かる。
  レポート_最終版_本物.doc             複数人が同時に作業しても
                                     混乱しない。
```

---

## GitとGitHubは別物

よくある混乱：

```
  Git    = 変更を追跡するツール（自分のコンピュータで動く）
  GitHub = Gitプロジェクトを保存・共有するウェブサイト

  Gitはカメラ。GitHubはオンラインで共有する写真アルバム。
```

## volta-auth-proxyでは

**volta-auth-proxyでは：** voltaのソースコードはすべてGitで追跡されており、今まで行われたすべての変更が記録・確認・取り消し可能です。何か問題が起きても、いつでも動作するバージョンに戻れます。
