# HTTPステータスコード（HTTP Status Codes）

[English version](http-status-codes.md)

---

## 一言で言うと？

HTTPステータスコードとは、サーバーがリクエストに対して「成功したか」「失敗したか」「他に何かすべきか」を伝えるために返す3桁の数字です。ウェイターが「お料理です」「その品は切れました」「先にお支払いをお願いします」と言うようなものです。

---

## レストランのたとえ

サーバーとやり取りするたびに（URLにアクセスしたり、APIを呼んだり）、サーバーはステータスコードで返答します。レストランでのウェイターの返答だと思ってください：

| コード範囲 | カテゴリ | ウェイターの発言... |
|---|---|---|
| **2xx** | 成功 | 「どうぞ！ごゆっくりどうぞ」 |
| **3xx** | リダイレクト | 「場所が変わりました。ご案内します」 |
| **4xx** | クライアントエラー（あなたのせい） | 「注文に問題があります」 |
| **5xx** | サーバーエラー（向こうのせい） | 「すみません、キッチンが火事です」 |

最初の桁で大まかなカテゴリが分かります。voltaで特に重要な具体的なコードを見ていきましょう。

---

## 200 OK -- 「お料理です」

**意味：** すべてうまくいきました。サーバーがリクエストを処理して、結果を返しています。

**voltaでは：** ページやAPIエンドポイントに正常にアクセスしたとき。例えば、voltaのForwardAuthチェックが通過して、アプリがページを提供するとき。

```
  GET /api/v1/users/me → 200 OK
  { "id": "taro-uuid", "email": "taro@acme.com" }

  レストラン：「パスタ1皿、ご注文通りです。どうぞ！」
```

---

## 301 Moved Permanently -- 「引っ越しました、アドレス帳を更新してください」

**意味：** 探しているものが恒久的に新しいURLに移動しました。ブラウザは今後新しいURLを使うべきです。

**voltaでは：** URL構造が恒久的に変わった場合。例えば`/login`が恒久的に`/auth/login`になった場合。

```
  GET /login → 301 Moved Permanently
  Location: /auth/login

  レストラン：「隣に恒久的に移転しました。
               GPSを更新してください」
```

---

## 302 Found（一時的リダイレクト）-- 「別のテーブルにご案内します」

**意味：** 求めているものが一時的に別のURLにあります。ブラウザは今回はそこに行くべきですが、将来は元のURLを使い続けてください。

**voltaでは：** これはvoltaで最もよくあるリダイレクトです。ログインしていない状態で保護されたページにアクセスすると、voltaがログインページにリダイレクトします。ログイン後、元のページに戻されます。

```
  GET /wiki/home → 302 Found
  Location: /login?redirect=/wiki/home

  レストラン：「テーブルの準備がまだです。
               バーでお待ちください、お呼びします」
```

---

## 400 Bad Request -- 「ご注文が理解できません」

**意味：** リクエストに何か問題があり、サーバーが処理できません。無効なデータを送ったか、必須フィールドが抜けているかもしれません。

**voltaでは：** API呼び出しで必須フィールドが欠けている場合や、無効なデータの場合。例えば、メールアドレスを指定せずに招待を作ろうとしたとき。

```
  POST /api/v1/tenants/acme-uuid/invitations
  Body: { "role": "MEMBER" }
  → 400 Bad Request: "email is required"

  レストラン：「『あのアレをください』と言われましても。
               具体的な料理名をお願いします」
```

---

## 401 Unauthorized -- 「どちら様ですか？身分証をお見せください」

**意味：** サーバーがあなたが誰か分かりません。まずログイン（認証）する必要があります。

**voltaでは：** ログインせずに保護されたリソースにアクセスしようとしたとき、またはセッションが期限切れのとき。

```
  GET /api/v1/users/me（セッションCookieなし）
  → 401 Unauthorized

  レストラン：「ここは会員制クラブです。
               会員証をお見せください」
```

注意：「Unauthorized」という名前にもかかわらず、このコードは実際には「認証されていない」（「認可されていない」ではない）を意味します。これはHTTP仕様の有名な命名ミスです。詳しくは[authentication-vs-authorization.md](authentication-vs-authorization.md)をご覧ください。

---

## 403 Forbidden -- 「あなたが誰かは分かりますが、それは提供できません」

**意味：** サーバーはあなたが誰か知っています（ログイン済み）が、このリソースへのアクセス権限がありません。

**voltaでは：** MEMBERが管理者専用ページにアクセスしようとしたとき、またはVIEWERが何かを編集しようとしたとき。

```
  GET /admin/settings（太郎がMEMBERとしてログイン中）
  → 403 Forbidden

  レストラン：「常連のお客様だとは存じていますが、
               このテーブルはVIP専用です」
```

---

## 404 Not Found -- 「メニューにございません」

**意味：** サーバーが探しているものを見つけられません。URLが存在しません。

**voltaでは：** 存在しないページやAPIエンドポイントにアクセスしようとしたとき。voltaはテナントやユーザーが見つからないときにも404を返します（リソースが存在するがアクセスできないことを明かさないため）。

```
  GET /api/v1/tenants/nonexistent-uuid
  → 404 Not Found

  レストラン：「当店ではお取り扱いしておりません。
               お店をお間違えではないですか？」
```

---

## 429 Too Many Requests -- 「注文が早すぎます、落ち着いてください」

**意味：** 短時間に多すぎるリクエストを送りました。サーバーが不正使用を防ぐためにレート制限しています。

**voltaでは：** 誰かがログインを何度も試みたとき（ブルートフォース攻撃の可能性）、またはAPIクライアントが多すぎるリクエストを送ったとき。

```
  POST /auth/login（1分間で10回目の試行）
  → 429 Too Many Requests
  Retry-After: 60

  レストラン：「立て続けに10回も注文を返されましたね。
               60秒お待ちいただいてから再度ご注文ください」
```

---

## 500 Internal Server Error -- 「キッチンが火事です」

**意味：** サーバー側で何か問題が起きました。あなたのせいではありません。サーバーにバグがあるか、予期しない状況に遭遇しました。

**voltaでは：** データベースエラー、設定ミス、またはハンドルされていない例外が発生したとき。500が出たら報告してください。バグです。

```
  GET /api/v1/users/me
  → 500 Internal Server Error

  レストラン：「大変申し訳ございません。シェフがお料理を
               床に落としてしまいました。すぐに対応いたします」
```

## クイックリファレンス表

| コード | 名前 | 誰のせい？ | voltaでの例 |
|---|---|---|---|
| 200 | OK | 誰のでもない（成功！） | ページ読み込み、APIがデータを返す |
| 301 | Moved Permanently | 誰のでもない（リダイレクト） | 古いURLが恒久的に変更 |
| 302 | Found（リダイレクト） | 誰のでもない（リダイレクト） | 「先にログインして、戻ってきて」 |
| 400 | Bad Request | クライアント（あなた） | リクエストに不足・無効なデータ |
| 401 | Unauthorized | クライアント（あなた） | 未ログイン、セッション期限切れ |
| 403 | Forbidden | クライアント（あなた） | ログイン済みだがロールが不適切 |
| 404 | Not Found | クライアント（あなた） | URLやリソースが存在しない |
| 429 | Too Many Requests | クライアント（あなた） | レート制限超過 |
| 500 | Internal Server Error | サーバー（volta） | バグや予期しないエラー |
