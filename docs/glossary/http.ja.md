## これは何？

**HTTPはウェブブラウザがウェブサイトと会話する方法です。手紙のやり取りのようですが、一瞬で届きます。**

知らなくても大丈夫です！HTTPはウェブサイトを訪れるたびに自動的に行われています。知らないうちに毎日使っています。

---

## 身近なたとえ

レストランにいることを想像してください：

```
  あなた（ブラウザ）：「メニューを見せてください」     --> GETリクエスト
  ウェイター（サーバー）：「メニューはこちらです！」   --> レスポンス

  あなた：               「ピザをお願いします」        --> POSTリクエスト
  ウェイター：           「かしこまりました！」        --> レスポンス
```

HTTPはまさにこのように動きます。ブラウザがサーバーに何かを要求し（ウェブページ、画像、データ）、サーバーがそれを返します。すべてのやり取りは**リクエスト**とそれに続く**レスポンス**です。

---

## 最もよく使う2つのタイプ

### GET -- 「何かをください」

ウェブサイトのアドレスを入力してEnterを押すと、ブラウザは**GET**リクエストを送ります。「このページをください」とお願いしています。

```
  入力: https://example.com/about

  ブラウザが送る:
  GET /about

  サーバーが返す:
  「Aboutページをどうぞ！」 (200 OK)
```

GETは**読み取り**用です。何も変えず、見ているだけです。

### POST -- 「これを受け取ってください」

フォームに入力して送信ボタンをクリックすると、ブラウザは**POST**リクエストを送ります。「これが私の情報です。処理してください」と伝えています。

```
  ログインフォームに入力して「サインイン」をクリック

  ブラウザが送る:
  POST /login
  username=tanaka&password=****

  サーバーが返す:
  「ようこそ！ログインしました。」 (200 OK)
```

POSTは**データを送る**ためのものです。何かを変えています（ログイン、注文、フォームの送信）。

---

## ステータスコード -- 簡単な説明

サーバーが応答するとき、結果を示す番号がつきます：

```
  200  =  「すべて問題なし！」（OK）
  301  =  「このページは移動しました。あちらをどうぞ。」
  404  =  「お探しのものが見つかりません。」（Not Found）
  500  =  「こちら側で何か壊れました、すみません。」（Server Error）
```

「404 Not Found」を見たことがあるでしょう。これでその意味が分かりましたね！

## volta-auth-proxyでは

**volta-auth-proxyでは：** すべてのログイン試行、すべてのページ表示、すべてのトークンチェックはHTTPで行われます。パスワードを入力して「サインイン」をクリックすると、ブラウザがvoltaサーバーにHTTP POSTリクエストを送り、サーバーが認証情報を確認して応答します。
