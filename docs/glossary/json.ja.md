# JSON

[English version](json.md)

---

## これは何？

**JSONは、人間にもコンピュータにも読める形で情報を書き記す方法です。標準化された記入用紙のようなものです。**

知らなくても大丈夫です！JSONはインターネット上のいたるところにあり、パターンを覚えればとても簡単に読めます。

---

## 身近なたとえ

病院に行くと、問診票を書きますよね：

```
  +-------------------------------+
  | 患者情報                       |
  |-------------------------------|
  | 名前：   田中ゆき              |
  | 年齢：   28                    |
  | アレルギー：ピーナッツ、ほこり    |
  | 喫煙：   いいえ                |
  +-------------------------------+
```

JSONはこの問診票のようなものですが、コンピュータにも読める形で書かれています。上の問診票をJSONで書くとこうなります：

```json
{
  "name": "田中ゆき",
  "age": 28,
  "allergies": ["ピーナッツ", "ほこり"],
  "smoker": false
}
```

---

## ルールはシンプル

JSONのルールはたった数個です：

1. **名前は左側にクォーテーションで囲む**：`"name"`
2. **コロンで名前と値を区切る**：`"name": "ゆき"`
3. **文字の値はクォーテーションで囲む**：`"田中ゆき"`
4. **数字はクォーテーション不要**：`28`
5. **はい/いいえはtrue/falseと書く**：`false`
6. **リストは角括弧を使う**：`["ピーナッツ", "ほこり"]`
7. **全体を波括弧で囲む**：`{ ... }`

これだけです！もうJSONが読めます。

---

## なぜコンピュータはJSONを使うのか？

コンピュータは常にお互いに情報を送り合っています。そのためには：

- どこでも同じ（標準化されている）
- コンピュータが素早く読める
- 人間がチェックできるくらい読みやすい

このような形式が必要です。JSONがまさにそれです。スマートフォンで天気を確認すると、天気サービスがスマートフォンにJSONを送ります：

```json
{
  "city": "東京",
  "temperature": 22,
  "unit": "celsius",
  "forecast": "sunny"
}
```

スマートフォンがこれを、あなたが見るきれいな天気画面に変換します。

## volta-auth-proxyでは

**volta-auth-proxyでは：** JSONはシステム全体で使われています。ログイン済みであることを証明するトークンはJSON、APIのレスポンスはJSON、サービス間の設定データのやり取りもJSONです。
