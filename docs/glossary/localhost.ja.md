# localhost

[English version](localhost.md)

---

## これは何？

**localhostは、自分のコンピュータがウェブサイトのふりをしていることです。インターネットを使わずに、自分のマシンで動いているウェブページにアクセスするためのアドレスです。**

知らなくても大丈夫です！localhostは主に開発者がソフトウェアを作ってテストするとき、世界に公開する前に使うものです。

---

## 身近なたとえ

新しいレシピを開発するシェフだと想像してください：

```
  自宅のキッチンで試作する  =  localhost
  レストランを開く         =  本番サーバーにデプロイする
```

初めて作った料理をいきなり100人のお客さんに食べてもらうことはしませんよね！まず自宅で作って、自分で味見して、味付けを調整して、完璧になってからレストランで出します。

**localhostはソフトウェアの自宅キッチンです。** 自分のコンピュータでプログラムを動かし、テストして、問題を直し、それから他の人が使える本番サーバーに置きます。

---

## アドレス：127.0.0.1

すべてのコンピュータには「自分自身」を意味する特別なアドレスがあります：

```
  localhost = 127.0.0.1 = 「今ここにあるこのコンピュータ」
```

この2つは同じものです。「自分」を言い方を変えているだけです：

```
  http://localhost:7070     -- あなたのコンピュータで動くvolta
  http://127.0.0.1:7070    -- まったく同じもの！
```

どのコンピュータを使っていても、`localhost`は常にそのコンピュータ自身を指します。インターネットには出ていきません。「うちの家」と言うようなもので、誰が言っても自分の家を指します。

---

## なぜ最初にlocalhostでテストするのか？

```
  ┌─────────────────────────────────────────────────────┐
  │                                                     │
  │  ステップ1：localhostで作ってテスト                    │
  │           （自分だけ、失敗しても安全）                  │
  │                    │                                │
  │                    ▼                                │
  │  ステップ2：ステージングサーバーでチームに見せる          │
  │           （少人数、まだ非公開）                       │
  │                    │                                │
  │                    ▼                                │
  │  ステップ3：本番にデプロイ                             │
  │           （みんなが使える！）                         │
  │                                                     │
  └─────────────────────────────────────────────────────┘
```

localhostでテストするのが安全な理由：
- 自分だけが見れる（他の人はあなたのlocalhostにアクセスできない）
- 失敗しても誰にも影響しない
- インターネット接続が不要
- 高速（ネットワークの遅延がない）

## volta-auth-proxyでは

**volta-auth-proxyでは：** 開発中はvoltaをlocalhost（通常は`http://localhost:7070`）で実行します。これにより、本番サーバーにデプロイする前に、自分のコンピュータでログインフロー、ページデザイン、セキュリティ機能をテストできます。
