# ログアウト

[English version](logout.md)

---

## 一言で言うと？

ログアウトとは、認証済みの[セッション](session.ja.md)を終了し、システムにサインイン済みのユーザーとして認識されなくする行為です。

---

## ホテルのチェックアウト

[ログイン](login.ja.md)がホテルのチェックインなら、ログアウトはチェックアウトです：

| ホテル | ウェブ |
|---|---|
| 部屋の鍵を返す | ブラウザが[セッション](session.ja.md) [Cookie](cookie.ja.md) を削除 |
| フロントが鍵を無効化 | サーバーがデータベースからセッションを削除 |
| 鍵はもうドアを開けない | Cookie はもうアクセスを許可しない |
| 部屋が次のお客のために清掃される | サーバーリソースが解放される |

重要なのは：**両方が対応する必要がある**こと。鍵を捨てただけでホテルが無効化しなければ、拾った人がまだ部屋を開けられます。だからこそ適切なログアウトにはサーバー側でのセッション無効化が必要です。

---

## なぜ必要なの？

ログアウトがなければ：

- セッションが永久に生き続ける（有効期限まで）-- 共有パソコンでは危険
- [Cookie](cookie.ja.md) を盗まれてもアクセスを取り消せない
- コンプライアンス規制（GDPR、SOC 2）はセッション終了機能を要求
- 組織を離れたユーザーがセッション期限切れまでログイン状態のまま
- デバイスが盗まれた場合に「すべての場所からサインアウト」ができない

## volta-auth-proxy でのログアウト

volta はサーバー側のセッション破棄を実装しています：

```
  ブラウザ                   volta-auth-proxy              データベース
  ──────                    ──────────────────            ──────────
  1. 「ログアウト」をクリック
  2. POST /auth/logout ─────>
                             3. Cookie からセッション ID を読み取り
                             4. sessions テーブルからセッションを DELETE ──>
                             5. __volta_session Cookie をクリア
                               （Max-Age=0 で Set-Cookie）
  <────── 302 リダイレクト /
  6. ユーザーにログインページが表示
```

ログアウト時に volta がすること：

1. **PostgreSQL からセッションを削除** -- セッション ID が即座に無効になる。Cookie の値をコピーしていても無意味。
2. **Cookie をクリア** -- `__volta_session` を `Max-Age=0` で設定し、[ブラウザ](browser.ja.md)に削除を指示。
3. **ランディングページにリダイレクト** -- ユーザーに未認証状態を表示。

volta がしないこと：

- **Google セッションの取り消しはしない** -- volta からログアウトしても Google からはログアウトしない。これは意図的：他のサービスで Google を使っている可能性がある。
- **発行済み [JWT](jwt.ja.md) の取り消しはしない** -- JWT は有効期限（5分）まで有効。JWT の有効期限が短い理由がこれ -- ログアウト後の窓を限定するため。

## 具体的な例

volta で保護されたアプリからログアウトする流れ：

1. ユーザーがアプリ UI で「ログアウト」をクリック
2. アプリが `POST https://auth.acme.example.com/auth/logout` を呼び出す
3. volta が `__volta_session` Cookie を読み取る：`550e8400-e29b-41d4-...`
4. volta が `DELETE FROM sessions WHERE id = '550e8400-e29b-41d4-...'` を実行
5. volta がレスポンス[ヘッダー](header.ja.md)を設定：`Set-Cookie: __volta_session=; Path=/; HttpOnly; Secure; SameSite=Lax; Max-Age=0`
6. volta が HTTP 302 [リダイレクト](redirect.ja.md)で `/` に応答
7. [ブラウザ](browser.ja.md)が Cookie を削除しリダイレクトに従う
8. ユーザーにログインページが表示される
9. アプリがキャッシュした [JWT](jwt.ja.md) があれば、最大あと5分だけ有効、その後 401 で失敗
10. volta-sdk-js が 401 を検出、リフレッシュを試行、拒否される（セッションなし）、ログインにリダイレクト
