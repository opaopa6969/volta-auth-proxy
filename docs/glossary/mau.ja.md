# MAU（月間アクティブユーザー）

[English version](mau.md)

---

## これは何？

MAUはMonthly Active Users（月間アクティブユーザー）の略で、1か月間に少なくとも1回アプリケーションを実際に使った一意のユーザー数のことです。

スポーツジムの会員数で考えてみましょう。ジムには5,000人の登録会員がいるかもしれませんが、今月実際にカードをスワイプして来館したのは1,200人だけです。この1,200人が「月間アクティブユーザー」です。ジムは総会員数（5,000人）で家賃を請求することもできますし、実際の来館者数（1,200人）で請求することもできます。SaaS の世界では、ほとんどの認証プロバイダがMAU -- つまり実際にやってきた人の数 -- に基づいて課金します。

---

## なぜ重要なのか？

MAUは、Auth0、Clerk、Oktaなどのクラウドホスト型認証サービスで最も一般的な課金指標です。月間にログインするユーザーが増えれば増えるほど、支払いが増えます。

問題はこうです：MAUベースの料金は最初は安く感じますが、驚くようなスケールの仕方をします。

### コストカーブ

| MAU | Auth0（概算） | Clerk（概算） | volta-auth-proxy |
|-----|--------------|--------------|------------------|
| 1,000 | 無料枠 | 無料枠 | $0 |
| 10,000 | ~$230/月 | ~$250/月 | $0 |
| 50,000 | ~$1,150/月 | ~$1,000/月 | $0 |
| 100,000 | ~$2,400/月 | ~$2,000/月 | $0 |
| 500,000 | 個別見積り（$$$$） | 個別見積り（$$$$） | $0 |

SaaSが成長し始めると、毎月の認証費用がインフラコストの中で最大になることがあります。データベースより高く、サーバーより高くなるのです。しかも[ベンダーロックイン](vendor-lock-in.ja.md)のため、簡単にプロバイダを切り替えることができません。

### MAUのカウント方法

サービスによってMAUのカウント方法が異なり、比較が難しくなっています：

- **Auth0：** ログインするか、トークンがリフレッシュされた場合に「アクティブ」とカウント。バックグラウンドでのサイレントトークンリフレッシュもカウントされます。
- **Clerk：** 課金期間中に少なくとも1回認証したユーザーをカウント。
- **Okta：** アクティブなユーザーだけでなく、プロビジョニングされた（システムに存在する）ユーザーをカウント -- これはさらにたちが悪いです。

重要なポイント：自分のMAU数は思っているほどコントロールできないことが多いです。バックグラウンドのトークンリフレッシュ、自動化システム、テストアカウントもすべてカウントされます。

---

## volta-auth-proxyではどうなっているか？

volta-auth-proxyは[セルフホスト](self-hosting.ja.md)です。自分のサーバーで実行します。ユーザー単位の課金もMAU計測もなく、利用状況を外部に送信する仕組みもありません。

ユーザーが100人でも1,000,000人でも、voltaの実行コストは自分のインフラのコスト（小さなPostgreSQLデータベースと約30MBのRAMを使う軽量なJavaプロセス）だけです。

```
  Auth0の料金：
  ┌──────────────────────────────────────┐
  │  MAUあたり$0.02 ... × 100,000       │
  │  = 認証だけで月$2,000               │
  └──────────────────────────────────────┘

  volta-auth-proxyの料金：
  ┌──────────────────────────────────────┐
  │  小さなVM 1台 + Postgres             │
  │  = 合計月~$20-50                     │
  │  （100人でも1,000,000人でも同じ料金）  │
  └──────────────────────────────────────┘
```

これがvoltaが存在する根本的な理由の一つです。SaaSが成長したとき、認証が最も高額な費用項目になるべきではありません。

## 現実の例

プロジェクト管理SaaSを構築しているとします。1年目は2,000MAU。Auth0の無料枠でカバーできます。快適です。

2年目は30,000MAUに成長。Auth0の請求は月~$700。理想的ではないですが、なんとかなります。

3年目は150,000MAU。Auth0は月$3,000以上。年間$36,000がログイン機能だけにかかります。切り替えたいのですが、コードベース全体がAuth0のSDK、Auth0のユーザーデータベース、Auth0のWebhookフォーマットを使っています。切り替えには何か月ものエンジニアリング作業が必要です。

これがMAUの罠です。voltaはセルフホストで、ユーザー数に関係なく無料なので、これを完全に回避します。
