# Maven

[English version](maven.md)

---

## これは何？

**MavenはJavaプロジェクトのレシピ本兼材料調達係です。プロジェクトに必要な材料（ライブラリ）を把握して、自動的にお店に買いに行ってくれます。**

知らなくても大丈夫です！MavenはJava開発者が使うツールです。Javaのコードを書いていなければ、ほとんど出会うことはありません。

---

## 身近なたとえ

豪華なケーキを焼きたいとします。2つの選択肢があります：

**Mavenなし：**
```
  1. レシピを読む
  2. 小麦粉を買いにA店へ
  3. 特別なチョコレートを買いにB店へ（A店になかった）
  4. バニラエッセンスを買いにC店へ
  5. チョコレートに特定のバターが必要と気づく
  6. バターを買いにA店に戻る
  7. やっと焼き始める
  8. 小麦粉のバージョンが間違っていたと気づく
  9. 最初からやり直し...
```

**Mavenあり：**
```
  1. 必要なものをリストに書く（pom.xml）
  2. Mavenが全部のお店に行ってくれる
  3. Mavenが全部正しいバージョンを持ってくる
  4. Mavenは材料に必要な材料まで持ってきてくれる
  5. 焼き始める！
```

---

## pom.xmlファイル

`pom.xml`ファイルはMavenのレシピカードです。プロジェクトに必要なものすべてを書きます：

```xml
<!-- これは簡略化したpom.xmlです -->
<project>
  <name>volta-auth-proxy</name>

  <!-- 材料（依存関係） -->
  <dependencies>
    <!-- Webサーバーのライブラリが必要 -->
    <dependency>
      <groupId>io.javalin</groupId>
      <artifactId>javalin</artifactId>
      <version>6.1.0</version>
    </dependency>

    <!-- データベースのライブラリが必要 -->
    <dependency>
      <groupId>org.postgresql</groupId>
      <artifactId>postgresql</artifactId>
      <version>42.7.0</version>
    </dependency>
  </dependencies>
</project>
```

細かいことは気にしないでください！ポイントは：プロジェクトに必要な「材料」のリストで、Mavenがすべて自動的に取ってきてくれるということです。

---

## 「依存関係（dependencies）」とは？

**依存関係（dependency）**は単なる材料です。プロジェクトが動くために必要なソフトウェアの部品です。ケーキに小麦粉、卵、砂糖が必要なように、Javaプロジェクトには以下のようなライブラリが必要です：

- データベースとの通信用
- Webサーバーの実行用
- セキュリティの処理用

そして小麦粉に小麦が必要（小麦には水と土が必要）なように、依存関係には別の依存関係が必要なこともあります。Mavenがこれをすべて自動で処理します。

---

## Maven vs Gradle

「Gradle」という名前を聞くかもしれません。Mavenと同じ仕事をする別のツールです：

```
  Maven  = 特定のフォーマットの信頼できるレシピ本
  Gradle = 別のフォーマットのより柔軟なレシピ本
```

どちらでも大丈夫です。voltaはよりシンプルで広く理解されているMavenを使っています。

---

## volta-auth-proxyでは

**volta-auth-proxyでは：** Mavenがプロジェクトのすべてのjava依存関係を管理しています。voltaが必要とするライブラリ（Webサーバー、データベースドライバー、セキュリティツール、テンプレートエンジン）を自動でダウンロードし、プロジェクトを実行可能なアプリケーションにビルドします。
