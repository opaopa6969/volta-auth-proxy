# ネットワーク

[English version](network.md)

---

## 一言で言うと？

ネットワークとは、データをやり取りできるように接続されたコンピュータの集まりです。家庭の Wi-Fi からインターネット全体まで含みます。

---

## 道路でつながった住宅街

コンピュータを家だと考えてください。ネットワークはそれらをつなぐ道路です：

- **家庭ネットワーク** -- 3〜4軒の家（スマホ、ノートPC、スマートTV）が1台のルーターでつながる袋小路
- **オフィスネットワーク** -- 数十軒の家（ワークステーション、サーバー、プリンター）がスイッチでつながる住宅街
- **インターネット** -- 何百万もの住宅街をつなぐ国全体の道路網

| 現実世界 | ネットワーク |
|---|---|
| 家 | コンピュータ、スマホ、[サーバー](server.ja.md) |
| 住所 | IP アドレス（例: `192.168.1.5`） |
| 家をつなぐ道路 | イーサネットケーブルや Wi-Fi 信号 |
| 都市をつなぐ高速道路 | インターネットバックボーン（光ファイバー） |
| 交通ルール | [プロトコル](protocol.ja.md)（TCP/IP, [HTTP](http.ja.md)） |
| ゲーテッドコミュニティ | [ネットワーク分離](network-isolation.ja.md) |

---

## なぜ必要なの？

ネットワークがなければ：

- すべてのコンピュータが孤島 -- メールもウェブもクラウドサービスもない
- ファイル共有には USB メモリを物理的に持ち運ぶ必要がある
- [サーバー](server.ja.md)が誰にもウェブページを提供できない
- [ブラウザ](browser.ja.md)が接続する先がない
- 現代の SaaS アプリケーションは存在し得ない

ネットワークは、他のすべてが走る基盤です。[HTTP](http.ja.md)、[Cookie](cookie.ja.md)、[セッション](session.ja.md)、[JWT](jwt.ja.md) -- すべてネットワーク上を流れます。

## volta-auth-proxy でのネットワーク

volta は複数のネットワーク境界をまたいで動作します：

```
  インターネット（パブリックネットワーク）
  ┌──────────────────────────────────────────────┐
  │  ユーザーのブラウザ                             │
  │  ↕ HTTPS（SSL/TLS で暗号化）                   │
  │  リバースプロキシ（Traefik/Nginx）               │
  └──────────────────────────────────────────────┘
           │
  Docker ネットワーク（プライベート/分離）
  ┌──────────────────────────────────────────────┐
  │  リバースプロキシ ←→ volta-auth-proxy          │
  │  リバースプロキシ ←→ あなたのアプリ              │
  │  volta-auth-proxy ←→ PostgreSQL              │
  │  volta-auth-proxy ←→ Google（OIDC 用）        │
  └──────────────────────────────────────────────┘
```

volta でのネットワークの主なポイント：

- **パブリック vs プライベート** -- [ブラウザ](browser.ja.md)はパブリックインターネット経由でリバースプロキシと通信。volta-auth-proxy と PostgreSQL はインターネットから直接到達できないプライベート [Docker](docker.ja.md) ネットワーク上に存在。
- **[ネットワーク分離](network-isolation.ja.md)** -- volta-auth-proxy はインターネットから直接アクセスできてはいけない。リバースプロキシだけがアクセスできるべき。
- **[ポート](port.ja.md)の公開** -- volta は[ポート](port.ja.md)（例: 8080）で待ち受けるが、そのポートは Docker ネットワーク内だけに公開され、外部には公開されない。
- **[ForwardAuth](forwardauth.ja.md)** -- このパターンはプライベートネットワーク内で完結する。リバースプロキシがリクエストをアプリに転送する前に、内部ネットワーク経由で volta に問い合わせる。

---

## 具体的な例

ユーザーが volta で保護されたアプリにアクセスするときのネットワークフロー：

1. ユーザーの[ブラウザ](browser.ja.md)は自宅の Wi-Fi ネットワーク上にある
2. ブラウザがインターネット経由で `app.acme.example.com` に HTTPS リクエストを送信
3. DNS が[ドメイン](domain.ja.md)をサーバーのパブリック IP（例: `203.0.113.10`）に解決
4. リクエストが[リバースプロキシ](reverse-proxy.ja.md)の[ポート](port.ja.md) 443 に到着
5. リバースプロキシはパブリックネットワークとプライベート Docker ネットワークの両方に接続
6. リバースプロキシが `http://volta:8080/auth/verify` に内部リクエスト（プライベートネットワーク、[SSL/TLS](ssl-tls.ja.md) 不要）
7. volta が `postgres:5432` で PostgreSQL に問い合わせ（同じくプライベートネットワーク）
8. volta が認証[ヘッダー](header.ja.md)付きでリバースプロキシに応答
9. リバースプロキシが元のリクエストを `http://app:3000` のアプリに転送（プライベートネットワーク）
10. レスポンスがリバースプロキシ経由でインターネットを通ってユーザーのブラウザに戻る

ステップ6〜9はすべてプライベートネットワーク内で完結。[ブラウザ](browser.ja.md)は volta と直接通信しません。

## さらに学ぶために

- [ネットワーク分離](network-isolation.ja.md) -- どのコンピュータが互いに通信できるかを制限する
- [プロトコル](protocol.ja.md) -- ネットワーク上でコンピュータが通信するルール
- [SSL/TLS](ssl-tls.ja.md) -- ネットワーク上を流れるデータの暗号化
- [ポート](port.ja.md) -- ネットワーク接続されたコンピュータの番号付きドア
- [Docker](docker.ja.md) -- volta がサービス用に分離されたネットワークを作る方法
- [ForwardAuth](forwardauth.ja.md) -- プライベートネットワーク内で行われる認証
