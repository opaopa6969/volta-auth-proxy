# OIDCプロバイダ

[English version](oidc-provider.md)

---

## これは何？

OIDCプロバイダ（OpenIDプロバイダ、OPとも呼ばれる）は、人の身元を確認し、その確認の証明を他のアプリケーションに発行するサービスです。

公証人のようなものです。法的書類のために自分が誰かを証明する必要があるとき、公証人のところに行きます。公証人があなたの身分証を確認し、あなたが名乗る通りの人物であることを確認し、書類に公証の印を押します。他の関係者は公証人の印を信頼し、自分であなたの身元を確認する必要がありません。OIDCプロバイダはインターネットの公証人です -- あなたの身元を（通常パスワードや生体認証で）確認し、「はい、この人は本人です」という署名付きトークンを渡します。

---

## なぜ重要なのか？

OIDCプロバイダがなければ、すべてのアプリケーションがパスワード付きの独自ユーザーデータベースを管理しなければなりません。つまり：

- すべてのアプリがパスワードを安全に保存しなければならない（多くが正しくできない）
- ユーザーはすべてのサービスで新しいアカウントを作る必要がある
- ユーザーはサイト間でパスワードを使い回す（大きなセキュリティリスク）
- 各アプリがパスワード漏洩の標的になりうる

OIDCプロバイダはIDを一元化することでこれを解決します。たとえばGoogleはすでにあなたが誰か知っています。アプリがユーザーにパスワードを求める代わりに、Googleに聞けます：「この人が本物か確認してくれる？」Googleがパスワード（またはパスキー、2FA）を処理し、署名付きの確認を返します。

## 一般的なOIDCプロバイダ

| プロバイダ | タイプ | 使用者 |
|-----------|--------|--------|
| **Google** | パブリック（誰でも利用可能） | コンシューマーアプリ、SaaS |
| **Microsoft（Azure AD）** | パブリック + エンタープライズ | Office 365、企業アプリ |
| **Apple** | パブリック | iOSアプリ、コンシューマーサービス |
| **Okta** | エンタープライズ | 企業の従業員ID管理 |
| **Keycloak** | セルフホスト | コントロールしたい組織 |
| **Auth0** | クラウドホスト | 利便性を求める開発者 |

volta-auth-proxyは**Google**をOIDCプロバイダとして使います。ユーザーがvoltaのログインページで「Googleでサインイン」をクリックすると、以下が起きます：

```
  ユーザーが                volta-auth-proxy         Google（OIDCプロバイダ）
  「Googleでサインイン」
  をクリック
  ─────────────────────►
                             1. state、nonce、PKCEを生成
                             2. ユーザーをGoogleにリダイレクト
                             ─────────────────────────────►

                                                      3. Googleがログインページを表示
                                                      4. ユーザーがパスワード/パスキーを入力
                                                      5. Googleが身元を確認
                                                      6. Googleがコード付きでリダイレクトバック

                             ◄─────────────────────────────
                             7. コードをトークンと交換
                             ─────────────────────────────►

                                                      8. Googleが返す：
                                                         - id_token（この人は誰か）
                                                         - access_token

                             ◄─────────────────────────────
                             9. id_tokenの署名を検証
                             10. メール、名前、写真を抽出
                             11. DBでユーザーを作成/更新
                             12. セッションを作成
  ◄─────────────────────
  ログイン完了！
```

このフローでは、GoogleがOIDCプロバイダです。voltaはユーザーのGoogleパスワードを一切見ません。voltaはGoogleの署名付きid_tokenを身元の証明として信頼します。

## 重要な概念

### ディスカバリドキュメント

すべてのOIDCプロバイダはよく知られたURLで「ディスカバリドキュメント」を公開します。これにより他のシステムがプロバイダとの通信方法を知ります：

```
Googleのディスカバリドキュメント：
https://accounts.google.com/.well-known/openid-configuration

内容：
{
  "issuer": "https://accounts.google.com",
  "authorization_endpoint": "https://accounts.google.com/o/oauth2/v2/auth",
  "token_endpoint": "https://oauth2.googleapis.com/token",
  "jwks_uri": "https://www.googleapis.com/oauth2/v3/certs",
  ...
}
```

voltaはGoogleのディスカバリドキュメントを読んで、ユーザーをどこにログインに送るか、認可コードをどこでトークンと交換するかを知ります。

### id_token

id_tokenはOIDCプロバイダが署名する[JWT](jwt.ja.md)で、「この人の身元を確認しました」と言うものです。以下のようなクレームを含みます：

```json
{
  "iss": "https://accounts.google.com",
  "sub": "1234567890",
  "email": "taro@gmail.com",
  "name": "Taro Yamada",
  "picture": "https://...",
  "email_verified": true,
  "exp": 1711900000
}
```

voltaはGoogleの公開鍵（`jwks_uri`から取得）を使ってこのトークンの署名を検証し、メールと名前を使ってユーザーを作成または検索します。

## OIDCプロバイダの「消費者」としてのvolta

voltaはOIDCプロバイダ自体ではありません。voltaはOIDCの**消費者**（リライングパーティ、RPとも呼ばれる）です。Google（プロバイダ）がユーザーを確認することを信頼し、その後voltaが残りすべてを処理します：セッション、マルチテナンシー、ロール、下流アプリ向けのJWT。

```
  IDチェーン：

  Google（OIDCプロバイダ）
    │ 「この人は taro@gmail.com です」
    ▼
  volta-auth-proxy（OIDC消費者 / リライングパーティ）
    │ 「taro@gmail.com はACME Corpテナントに属し、ADMINロールを持つ」
    ▼
  あなたのアプリ（voltaヘッダーを読む）
    │ 「ACME Corpの管理ダッシュボードを表示」
    ▼
  ユーザーがダッシュボードを見る
```

GoogleがIDを認証。voltaがビジネスコンテキスト（テナント、ロール）を追加。あなたのアプリはヘッダーを読むだけ。

---

## voltaは他のOIDCプロバイダでも動くか？

voltaはGoogle OIDCを主要プロバイダとして構築されています。IdP設定（Phase 4）により、テナントごとに追加のOIDCプロバイダやSAMLアイデンティティプロバイダを設定できるようになり、エンタープライズ顧客は自社の企業IDシステムを使えます。
