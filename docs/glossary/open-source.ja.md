# オープンソース

[English version](open-source.md)

---

## これは何？

**オープンソースソフトウェアは、誰でもレシピを見られるソフトウェアです。作り方の指示（コード）が公開されていて、無料で使え、自由に改変できます。**

知らなくても大丈夫です！オープンソースは技術ではなく考え方です。アイデアを理解すれば、とてもシンプルです。

---

## 身近なたとえ

レシピについて考えてみましょう：

```
  クローズドソース（独占的）：         オープンソース：

  ┌─────────────────────┐             ┌─────────────────────┐
  │  秘密のレシピ         │             │  公開レシピ           │
  │                      │             │                      │
  │  コカ・コーラの配合は  │             │  おばあちゃんの       │
  │  金庫に入っている。    │             │  クッキーレシピ。     │
  │  飲み物は買えるけど、  │             │  みんなに教えて       │
  │  作り方は見れないし、  │             │  くれる。誰でも       │
  │  自分で作れない。      │             │  作れるし、変えたり   │
  │                      │             │  改良したりできる。    │
  └─────────────────────┘             └─────────────────────┘
```

オープンソースソフトウェアはおばあちゃんのクッキーレシピのようなものです。誰でも読めて、使えて、改変して、改良を他の人と共有できます。

## 「オープンソース」の実際の意味

| できること： | できないこと（通常）： |
|----------|----------------------|
| コードを全部読む | 自分が書いたと主張する（クレジットが必要） |
| 無料で使う | （ライセンスによっては特定のルールあり） |
| 自分のニーズに合わせて改変する | |
| 他の人と共有する | |
| コードから学ぶ | |

## なぜ無料でソフトウェアを配るのか？

いい質問です！理由はたくさんあります：

- **コミュニティの改善** -- 何千人もがバグを見つけて修正できるので、1つの会社だけより良いソフトウェアになる
- **信頼** -- コードが公開されていれば、安全かどうか誰でも確認できる（隠れたスパイ機能がない）
- **学習** -- 新人開発者が本物のコードを読んで学べる
- **恩返し** -- 多くの開発者がオープンソースの恩恵を受けて、他の人も助けたいと思っている
- **ビジネスモデル** -- 中核ソフトウェアは無料にして、サポートやホスティング、プレミアム機能で収益を得る企業も多い

## 有名なオープンソースプロジェクト

知らないうちにオープンソースを毎日使っているかもしれません：

```
  Linux       - インターネットのサーバーのほとんどを動かしている
  Firefox     - ウェブブラウザ
  Android     - スマートフォンのOS（Linuxベース）
  Wikipedia   - その裏のソフトウェアはオープンソース
  PostgreSQL  - データベース（voltaが使っている！）
  WordPress   - 全ウェブサイトの約40%を動かしている
```

---

## volta-auth-proxyでは

**volta-auth-proxyでは：** voltaの依存関係はすべてオープンソースです。Webサーバー（Javalin）、データベース（PostgreSQL）、テンプレートエンジン（jte）、その他多くのライブラリ。つまり、スタック全体を誰でも検査、信頼、改善できます。
