# ポート番号

[English version](port.md)

---

## これは何？

**ポート番号はマンションの部屋番号のようなものです。コンピュータの中のどのプログラムがメッセージを受け取るべきかを教えてくれます。**

知らなくても大丈夫です！ポート番号は裏側で動いていて、ほとんどの人は気にする必要がありません。

---

## 身近なたとえ

大きなマンションを想像してください：

```
  ┌─────────────────────────────────┐
  │  インターネット通り123番地         │  <-- これがコンピュータ
  │  （コンピュータのアドレス）        │     （IPアドレスのようなもの）
  │                                 │
  │   部屋80   - Webサーバー         │  <-- HTTP（ウェブページ）
  │   部屋443  - 安全なWebサーバー    │  <-- HTTPS（安全なウェブページ）
  │   部屋5432 - Postgresデータベース │  <-- データベース
  │   部屋7070 - volta-auth-proxy   │  <-- volta！
  │   部屋3000 - あなたのアプリ       │  <-- あなたのアプリケーション
  │                                 │
  └─────────────────────────────────┘
```

建物の住所（インターネット通り123番地）で正しい建物にたどり着き、部屋番号で正しいドアにたどり着きます。ポート番号も同じです。コンピュータ上の正しいプログラムにメッセージを届けます。

---

## なぜ2つのプログラムが同じポートを使えないのか？

2つの家族が同じ部屋に住めないように、2つのプログラムが同時に同じポート番号を使うことはできません。voltaがすでにポート7070を使っていたら、別のプログラムは7070を使えません。

使おうとすると、こんなエラーが出ます：

```
  「ポート7070はすでに使用中です！」
```

マンションの部屋に行ったら、もう別の人が住んでいた、というようなものです。

---

## よく使われるポート番号

有名な住所のように、みんなが知っているポート番号があります：

```
  ポート80    =  ウェブページ（HTTP）       -- 「メインストリート」のようなもの
  ポート443   =  安全なウェブページ（HTTPS） -- 「メインストリートのVIP入口」
  ポート5432  =  PostgreSQLデータベース     -- Postgresのデフォルト
  ポート3306  =  MySQLデータベース          -- 別のデータベース
  ポート22    =  SSH（リモートログイン）     -- サーバーに遠隔アクセスするため
```

---

## ポート番号をどこで見るか

ブラウザがよく使うポートを隠してくれるので、ポート番号を入力したことはないかもしれません：

```
  入力するもの：         https://example.com
  実際に起きること：     https://example.com:443  （ポート443が暗黙的に使われる）
```

でも開発中はよくポートを見かけます：

```
  http://localhost:7070   -- あなたのコンピュータで動くvolta（部屋7070）
  http://localhost:3000   -- あなたのコンピュータで動くアプリ（部屋3000）
```

---

## volta-auth-proxyでは

**volta-auth-proxyでは：** プロキシサーバーはポート7070で待ち受け、PostgreSQLデータベースはポート54329を使います。異なる「部屋番号」なので、お互いにも、コンピュータ上の他のプログラムとも衝突しません。
