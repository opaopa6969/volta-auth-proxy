# 公開鍵暗号

[English / 英語](public-key-cryptography.md)

---

## これは何？

公開鍵暗号（非対称暗号とも呼ばれる）は、数学的に関連づけられた2つの鍵を使う仕組みです。**公開鍵**は誰でも見られ、**秘密鍵**は所有者だけが秘密に保持します。公開鍵で暗号化されたデータは秘密鍵でしか復号できず、その逆も同様です。これにより根本的な問題が解決されます：事前に会って秘密を共有しなくても、安全に通信できるようになります。

---

## なぜ重要？

公開鍵暗号がなかった時代は、暗号化メッセージを送りたければ事前に共有鍵を交換する必要がありました。これが「鍵配送問題」です -- 鍵をどうやって安全に共有するのか？ 公開鍵暗号はこの問題を解消します。公開鍵を公に配布し、誰でもそれを使ってあなた宛のメッセージを暗号化できますが、復号できるのは秘密鍵を持つあなただけです。

認証の世界では、公開鍵暗号は**デジタル署名**を可能にします。サーバーが秘密鍵でデータに署名し、誰でも公開鍵で署名を検証できます。JWT の仕組みがまさにこれです：volta が RSA 秘密鍵でトークンに署名し、下流のサービスが JWKS エンドポイントの公開鍵で検証します。

---

## 簡単な例

郵便受けに例えてみましょう：

- **公開鍵** = 玄関の郵便投入口。誰でも手紙を入れられる
- **秘密鍵** = 郵便受けを開ける鍵。あなただけが持っている

**署名**（作成者の証明）は逆方向に働きます：

- あなたが手紙を書き、秘密鍵でスタンプを押す（署名）
- 誰でも公開鍵でスタンプを確認し、本当にあなたからの手紙か確かめられる

```
署名:    秘密鍵  +  データ  ->  署名
検証:    公開鍵  +  データ  +  署名  ->  有効 / 無効
```

## volta-auth-proxy での使い方

volta は JWT 署名に RSA 公開鍵暗号（2048ビット鍵）を使います：

- **秘密鍵**: `JwtService` が RS256 アルゴリズムで JWT に署名するために使用。データベースに暗号化して保存（[encryption-at-rest.md](encryption-at-rest.md) 参照）。サーバーの外には出ない
- **公開鍵**: `/.well-known/jwks.json` エンドポイントで JWK（JSON Web Key）として公開。下流のサービスはこれを取得して、共有秘密なしに volta の JWT を検証可能

```java
// 署名（volta だけができる）
jwt.sign(new RSASSASigner(rsaKey.toPrivateKey()));

// 検証（公開鍵を持っていれば誰でもできる）
JWSVerifier verifier = new RSASSAVerifier(rsaKey.toRSAPublicKey());
jwt.verify(verifier);
```

つまり、下流のアプリは volta の秘密鍵を必要としません。JWKS から公開鍵を取得して独立に検証するだけです。秘密鍵が漏洩した場合、volta は新しい鍵ペアにローテーションできます（[key-rotation.md](key-rotation.md) 参照）。下流のサービスとの調整は不要です。
