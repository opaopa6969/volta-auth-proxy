# リポジトリ

[English version](repository.md)

---

## これは何？

リポジトリ（よく「リポ」と省略される）は、コードやソフトウェアアーティファクトの保管場所です。一般的に2種類あります：**[git](git.md)リポジトリ**はソースコードとその完全なバージョン履歴を保存し、**パッケージリポジトリ**（Maven Centralなど）はプロジェクトが依存するコンパイル済み[ライブラリ](library.md)を保存します。

タイムマシン付きのファイルキャビネットに似ています。gitリポジトリはすべてのファイルのすべてのバージョンを保存するため、過去のどの時点にも戻れます。ドキュメントの現在のバージョンだけでなく、先週の火曜日のバージョンや、6ヶ月前のバージョンも取り出せるファイルキャビネットを想像してください。パッケージリポジトリはむしろ倉庫に似ています。誰でも注文（ダウンロード）できる完成品（コンパイル済みライブラリ）を保管しています。

「voltaリポをcloneして」と言われたら、volta-auth-proxyのソースコードを含むgitリポジトリをダウンロードすることを意味します。Mavenが依存関係をダウンロードするとき、パッケージリポジトリから取得しています。

---

## なぜ重要なのか？

リポジトリはソフトウェア開発の基盤です：

- **コラボレーション** -- 複数の開発者が共有gitリポジトリを通じて同じコードベースで作業
- **バージョン履歴** -- すべての変更が記録される。何かが壊れたら、何がいつ変わったかを正確に確認できる
- **依存関係管理** -- [Maven](maven.md)がMaven Central（パッケージリポジトリ）から[ライブラリ](library.md)をダウンロード。なければランダムなウェブサイトからJARファイルを手動ダウンロードすることになる
- **バックアップ** -- リモートgitリポジトリ（GitHub、GitLabなど）はコードのバックアップ。ラップトップが壊れてもコードは生き残る
- **デプロイ** -- リポジトリが唯一の真実の情報源。リポにあるものが[ビルド](build.md)されデプロイされる

## どう動くのか？

### Gitリポジトリ

gitリポジトリはファイルの時間経過に伴う変更を追跡します：

```
  ┌────────────────────────────────────────────────────────────┐
  │                  Gitリポジトリ                               │
  │                                                             │
  │  コミット履歴:                                               │
  │                                                             │
  │  abc1234  ← HEAD（最新）                                    │
  │  │  「ログインエンドポイントにレート制限を追加」              │
  │  │  変更: src/main/java/AuthMiddleware.java                │
  │  │                                                          │
  │  def5678                                                    │
  │  │  「JWT期限チェックを修正」                                │
  │  │  変更: src/main/java/JwtService.java                    │
  │  │                                                          │
  │  ghi9012                                                    │
  │  │  「初回コミット」                                        │
  │  │  追加: 全ファイル                                        │
  │  ▼                                                          │
  └────────────────────────────────────────────────────────────┘
```

### ローカル vs リモートリポジトリ

```
  あなたのマシン                         GitHub/GitLab
  ┌──────────────────┐                  ┌──────────────────┐
  │  ローカルリポ     │                  │  リモートリポ     │
  │  （完全なコピー） │   git push ───> │  （完全なコピー） │
  │                   │   <─── git pull  │                   │
  │  .git/ ディレクトリ│                  │  チームと共有     │
  │  作業ファイル     │                  │  CI/CDが読み取り  │
  └──────────────────┘                  └──────────────────┘

  両方とも完全な履歴を含む。
  どちらもオフラインで作業可能。
```

### パッケージリポジトリ（Maven Central）

```
  pom.xmlの記述:                    Maven Central
  HikariCP 5.1.0が必要             ┌────────────────────┐
           │                        │ com/zaxxer/         │
           │   mvn compile          │   HikariCP/         │
           └───────────────────────>│     5.1.0/          │
                                    │       HikariCP.jar  │
           <───────────────────────│       HikariCP.pom  │
           ダウンロード先:           │       チェックサム   │
           ~/.m2/repository/        └────────────────────┘
```

### volta-auth-proxyのリポジトリ構造

```
  volta-auth-proxy/              （gitリポジトリのルート）
  ├── .git/                      （gitメタデータ -- 触らない）
  ├── pom.xml                    （Mavenビルド設定 + 依存関係）
  ├── volta-config.yaml          （ランタイム設定）
  ├── Dockerfile                 （コンテナビルド手順）
  ├── docker-compose.yml         （ローカル開発環境）
  ├── src/
  │   ├── main/
  │   │   ├── java/              （Javaソースコード）
  │   │   ├── jte/               （jteテンプレート）
  │   │   └── resources/         （静的ファイル、SQLマイグレーション）
  │   └── test/
  │       └── java/              （テストソースコード）
  ├── docs/                      （ドキュメント）
  │   └── glossary/              （これを読んでいる場所）
  └── target/                    （ビルド出力 -- gitに入れない）
```

### リポジトリに入れないもの

| 除外対象 | 理由 | 除外方法 |
|---------|------|---------|
| `target/` | ビルド出力、ソースから再生成可能 | `.gitignore` |
| `.env` | 秘密情報を含む（パスワード、APIキー） | `.gitignore` |
| `node_modules/` | ダウンロード済み依存関係（あれば） | `.gitignore` |
| `~/.m2/repository/` | ローカルMavenキャッシュ | プロジェクトディレクトリ外 |

## volta-auth-proxy ではどう使われている？

### ソースコード用のGitリポジトリ

volta-auth-proxyのソースコードはgitリポジトリに存在します。開発者がローカルで作業するために[clone](clone.md)します：

```bash
git clone https://github.com/your-org/volta-auth-proxy.git
cd volta-auth-proxy
mvn package
java -jar target/volta-auth-proxy.jar
```

### 依存関係用のMavenリポジトリ

voltaのpom.xmlが依存関係を宣言し、Mavenがリポジトリから解決します：

```
  解決順序:
  1. ローカルキャッシュを確認（~/.m2/repository/）
  2. 見つからなければMaven Centralからダウンロード
  3. 将来のビルド用にローカルにキャッシュ
```

### ローカルMavenリポジトリキャッシュ

```
  ~/.m2/repository/
  ├── com/zaxxer/HikariCP/5.1.0/
  │   ├── HikariCP-5.1.0.jar
  │   └── HikariCP-5.1.0.pom
  ├── com/github/ben-manes/caffeine/caffeine/3.1.8/
  │   ├── caffeine-3.1.8.jar
  │   └── caffeine-3.1.8.pom
  └── ...
```

このキャッシュにより、以降のビルドはネットワークアクセスが不要です。最初のビルドですべてをダウンロードし、将来のビルドはキャッシュを使います。

## よくある間違いと攻撃

### 間違い1：リポジトリに秘密をコミットする

秘密情報（APIキー、データベースパスワード）をgitにコミットすると、ファイルを後のコミットで削除しても、履歴に永遠に残ります。秘密には[環境変数](environment-variable.md)を使い、コミットしないでください。

```bash
# 悪い例: パスワードがコードに入り、リポにコミット
String dbPassword = "s3cret123";

# 良い例: パスワードは環境から、リポには入れない
String dbPassword = System.getenv("DATABASE_PASSWORD");
```

### 間違い2：ビルドアーティファクトをコミットする

`target/`ディレクトリはコミットすべきではありません。ソースから再ビルドできる生成ファイルが含まれています。含めるとリポジトリが肥大化し、マージコンフリクトが発生します。

### 攻撃1：依存関係の混乱

攻撃者が内部パッケージと同じ名前の悪意あるパッケージを公開リポジトリに公開します。Mavenが公開リポジトリをプライベートリポジトリより先にチェックすると、攻撃者のパッケージをダウンロードしてしまいます。防御策：プライベートリポジトリを先にチェックするようMavenを設定し、pom.xmlに明示的なリポジトリ宣言を使う。

### 間違い3：.gitignoreを使わない

適切な`.gitignore`がないと、開発者がIDEファイル（`.idea/`、`.vscode/`）、OSファイル（`.DS_Store`）、秘密情報を誤ってコミットします。voltaの`.gitignore`はこれらを除外するよう設定されています。

### 攻撃2：リポジトリの侵害

攻撃者がgitリポジトリへの書き込みアクセスを得ると、ビルドされてデプロイされる悪意あるコードを注入できます。防御策：コードレビュー（プルリクエスト）を必須にし、mainブランチの保護を有効にし、署名付きコミットを使う。
