# レスポンシブWebデザイン

[English version](responsive.md)

---

## これは何？

レスポンシブWebデザインは、表示される画面に合わせてレイアウト、サイズ、コンテンツを自動的に適応させるWebページ構築のアプローチです -- 27インチのデスクトップモニター、ノートPC、タブレット、6インチの携帯電話画面のいずれでも。「モバイル版」と「デスクトップ版」を別々に作る代わりに、利用可能な画面スペースに対応する1つのサイトを構築します。

異なる形のグラスに注がれる水のようなものです。水（コンテンツ）は常に同じですが、入っているグラス（画面）の形を取ります。広いグラスでは水が広がり、細いグラスでは縦長になります。水は変わりません -- その配置だけが変わります。

レスポンシブデザインは3つのCSS技術で構成されています：**フルードグリッド**（固定ピクセルの代わりにパーセンテージを使用するレイアウト）、**フレキシブルな画像**（コンテナ内でスケールする画像）、そして**メディアクエリ**（特定の画面幅でのみ適用されるCSSルール）。

---

## なぜ重要なのか？

現在、Webトラフィックの60%以上がモバイルデバイスからのものです。ログインページ、ダッシュボード、管理パネルがスマートフォンで壊れて見えると、ユーザーはアプリケーションを信頼しません -- 特にセキュリティの認識が重要な認証ページでは。

volta-auth-proxy向けには：

- **ログインページ**はスマートフォンで動作する必要がある。従業員が個人のモバイルデバイスからログインする必要があるかもしれない。
- **同意/アカウント選択画面**はGoogle（voltaではなく）がレンダリングしますが、voltaのページへのリダイレクト後の画面は任意の画面サイズで正しく表示される必要がある。
- **管理ダッシュボード**はタブレットで使用可能である必要がある（移動中の管理向け）。
- **エラーページ**（セッション期限切れ、アクセス拒否）は任意のデバイスで読める必要がある。

認証ページでのモバイル体験が壊れていると、ユーザーはこう思います：「このサービスはプロフェッショナルでない。認証情報を預けて大丈夫か？」第一印象は重要で、多くのユーザーにとってログインページが第一印象そのものです。

## どう動くのか？

### モバイルファーストアプローチ

volta.cssは**モバイルファースト**アプローチを使用します：デフォルトのCSSルールは小さな画面向けに設計され、大きな画面にはメディアクエリで追加ルールを適用します。これがベストプラクティスとされる理由：

1. モバイルスタイルはよりシンプル（シングルカラム、スタック要素）
2. コンテンツの優先順位付けを強制される（小さな画面でユーザーが見るべきものは？）
3. モバイルデバイスはより制約があるため、先に最適化することで肥大化を防ぐ

```css
/* モバイルファースト：小さな画面のデフォルトスタイル */
.login-container {
    width: 100%;
    padding: 1rem;
}

/* タブレット以上 */
@media (min-width: 768px) {
    .login-container {
        width: 480px;
        margin: 0 auto;
        padding: 2rem;
    }
}

/* デスクトップ */
@media (min-width: 1024px) {
    .login-container {
        width: 520px;
        padding: 3rem;
    }
}
```

### viewportメタタグ

レスポンシブデザインをモバイルで機能させるには、HTMLの`<head>`にこれが必要です：

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

これがないと、モバイルブラウザは仮想の「デスクトップ」幅（通常980px）でページをレンダリングし、画面に合うように縮小するため、すべてが小さく読めなくなります。

### 主要なCSS技術

#### フルード・タイポグラフィ

```css
/* 固定フォントサイズの代わりに */
body {
    font-size: 16px; /* モバイルのベース */
}

@media (min-width: 768px) {
    body {
        font-size: 18px;
    }
}

/* またはclamp()でスムーズなスケーリング */
h1 {
    font-size: clamp(1.5rem, 2.5vw, 2.5rem);
}
```

#### フレキシブルコンテナ

```css
.card {
    width: 100%;
    max-width: 600px; /* 600pxより広くならない */
    margin: 0 auto;   /* 広い画面で中央寄せ */
}
```

### 一般的なブレークポイント

| ブレークポイント | 対象デバイス | voltaでの使用 |
|---------------|-----------|-------------|
| `< 480px` | 小型スマートフォン | ログインフォームが画面いっぱい、最小限のパディング |
| `480px - 768px` | 大型スマートフォン、小型タブレット | ログインフォーム中央配置、適度なパディング |
| `768px - 1024px` | タブレット、小型ノートPC | 管理サイドバーがハンバーガーメニューに |
| `> 1024px` | デスクトップ | フルレイアウト、サイドバー表示 |

## volta-auth-proxy ではどう使われている？

volta-auth-proxyのCSS（`volta.css`）は**モバイルファーストレスポンシブ**です。voltaがレンダリングするすべてのページ -- ログイン、エラー、ダッシュボード、管理画面 -- がユーザーの画面サイズに適応します。

### レスポンシブでなければならないページ

| ページ | モバイルが重要な理由 |
|--------|-------------------|
| **ログイン** | ユーザーがモバイルブラウザからログインする場合がある。Googleリダイレクトはvoltaのコールバックに戻り、フローを開始したデバイスでレンダリングされる。 |
| **エラーページ** | 「セッション期限切れ」や「アクセス拒否」はいつでも任意のデバイスで表示される可能性がある。 |
| **アカウント設定** | ユーザーがスマートフォンから設定変更やセッション取り消しが必要かもしれない。 |
| **管理ダッシュボード** | テナント管理者がデスクから離れている時にユーザーアクティビティの確認やアラート対応が必要かもしれない。 |

### volta.cssの設計原則

1. **モバイルファースト**：デフォルトスタイルはスマートフォン向け。メディアクエリで大きな画面に複雑さを追加。
2. **CSSフレームワークなし**：voltaはBootstrap、Tailwind、いかなるCSSフレームワークも使用しない。CSSは手書きで最小限、voltaの「すべての行を理解する」哲学に合致。
3. **セマンティックHTML**：jteテンプレートはセマンティックHTML（`<main>`、`<nav>`、`<section>`）を使用し、レスポンシブレイアウトとアクセシビリティを支援。
4. **システムフォント**：voltaはユーザーのシステムフォントスタックを使用し、モバイルのレンダリングを遅くする外部フォントダウンロードを避ける。

## よくある間違いと攻撃

### 間違い1：viewportメタタグを忘れる

`<meta name="viewport" content="width=device-width, initial-scale=1.0">`がないと、モバイルブラウザはデスクトップ幅でレンダリングしてズームアウトします。すべてが小さく表示されます。最もよくあるレスポンシブデザインの間違いです。

### 間違い2：固定幅を使用する

```css
/* 悪い例 -- 小さな画面で壊れる */
.login-form { width: 500px; }

/* 良い例 -- レスポンシブ */
.login-form { width: 100%; max-width: 500px; }
```

### 間違い3：実機でテストしない

ブラウザDevToolsは画面サイズをエミュレートしますが、タッチ動作、ビューポートの癖、フォントレンダリングはエミュレートしません。重要なページ（特にログイン）は、少なくとも1台の実際のiOSとAndroidデバイスで必ずテストしてください。

### 間違い4：モバイルで重要なコンテンツを隠す

一部の開発者はスペース節約のためにモバイルでセキュリティ警告やエラーの詳細を非表示にします。セキュリティ関連情報を隠してはいけません。ログインが失敗した場合、ユーザーはどの画面サイズでも理由を確認できる必要があります。

### 間違い5：タッチターゲットが小さすぎる

ログインボタン、「パスワードを忘れた」へのリンク、「Googleでログイン」は、快適なタッピングのために少なくとも44x44ピクセル（Appleの推奨）または48x48dp（Googleの推奨）である必要があります。小さなタッチターゲットはユーザーを苛立たせ、アクセシビリティを低下させます。

### 攻撃：モバイル固有のフィッシング

攻撃者はモバイル画面で正規のログインページのように見えるフィッシングページを作成します。デスクトップでは違いが見える場合があります（アドレスバーの間違ったURL、HTTPSインジケーターの欠如）。モバイルではアドレスバーが小さく、しばしば非表示です。voltaはブックマークベースのアクセスとログインページでの目立つドメイン表示を促進してこれを軽減します。
