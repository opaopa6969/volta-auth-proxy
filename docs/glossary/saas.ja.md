# SaaS（Software as a Service）

[English version](saas.md)

---

## 一言で言うと？

SaaSとは、コンピュータに何もインストールせずにインターネット（通常はWebブラウザ）を通じて使うソフトウェアのことです。作った会社があなたの代わりに運用してくれるので、ログインするだけで使えます。

---

## たとえ話：購入 vs レンタル

映画を見ることを考えてみてください。2つの選択肢があります：

- **DVDを買う** -- あなたが所有します。自分のDVDプレーヤーで再生します。ディスクに傷がついたら自分の問題です。新しいバージョンが欲しければ、もう一度買います。
- **Netflixに加入する** -- 何も所有しません。ログインして見るだけです。Netflixがサーバー、アップデート、ストレージを管理します。新機能が追加されれば、自動的に使えるようになります。

SaaSはソフトウェア版のNetflixモデルです。プログラムを買ってコンピュータにインストールする代わりに、ブラウザを開いて使います。ソフトウェア会社が裏側のすべてを管理します。

---

## あなたがすでに使っている例

おそらく毎日SaaSを使っていることに気づいていないでしょう：

| SaaS製品 | 置き換わったもの |
|---|---|
| **Gmail** | PCにインストールしたOutlook、ハードドライブにメールをダウンロード |
| **Google Docs** | PCにインストールしたMicrosoft Word、デスクトップにファイルを保存 |
| **Slack** | 会社のサーバーにインストールしたオフィスチャットソフト |
| **Notion** | 会社のハードウェアで動かすwikiアプリケーション |
| **GitHub** | 自分でセットアップして管理するGitサーバー |

パターンは同じです：自分でソフトウェアをインストールして管理する代わりに、誰かが運用してくれて、インターネットを通じてアクセスします。

---

## SaaSとインストール型ソフトウェアの違い

| | インストール型ソフトウェア | SaaS |
|---|---|---|
| **動作する場所** | あなたのPCまたはサーバー | 会社のサーバー（「クラウド」） |
| **アップデート** | 手動でダウンロードしてインストール | 自動的に行われ、リフレッシュするだけ |
| **データの保存場所** | あなたのハードドライブ | 会社のサーバー |
| **アクセス** | インストールしたPCからのみ | ブラウザがあればどのデバイスからでも |
| **支払い** | 一回限りの購入（通常） | 月額または年額のサブスクリプション |
| **セットアップ** | インストールして設定する | アカウントを作って使い始める |

## なぜSaaSで認証が重要なの？

ここがポイントです：SaaSでは、多くの異なる会社や人々が同じアプリケーションを使っています。そのため、認証（あなたが誰かを確認すること）と認可（あなたが何をできるかを決めること）が極めて重要になります。

もしGmailに適切な認証がなかったらどうなるでしょう。ブラウザを開いたら、誰か他の人のメールが見えてしまうかもしれません。それは大惨事です。

SaaSアプリケーションには以下が必要です：

1. **あなたが誰かを知ること** -- アプリがどのデータを表示すべきか分かるように、ログインが必要
2. **あなたがどの組織に属しているかを知ること** -- あなたの会社がSlackを使っているなら、他の会社のチャンネルではなく自分の会社のチャンネルが見えるべき
3. **あなたが何をしてよいかを知ること** -- インターンがCEOと同じ権限を持つべきではない
4. **顧客間でデータを完全に分離すること** -- 会社Aが会社Bのデータを絶対に見てはいけない

これがまさにvolta-auth-proxyがやっていることです。SaaSアプリケーションの前に立って、これらすべての懸念を処理するので、アプリは本来の目的に集中できます。

## volta-auth-proxyとSaaSの関係

SaaS製品を作るとき、volta-auth-proxyはフロントドアとして機能します：

```
  ユーザーがあなたのSaaSアプリにアクセス：

  ユーザーのブラウザ
       │
       ▼
  volta-auth-proxy ← 「あなたは誰？どの会社？どんな役割？」
       │
       │  （確認済み！リクエストに本人情報を追加）
       ▼
  あなたのアプリ ← 「ああ、ACME Corpの太郎さん、ADMINだ」
       │
       ▼
  太郎にACME Corpのデータだけを表示
```

voltaがなければ、アプリがこのログイン、テナント、ロールのロジックをすべてゼロから作る必要があります。voltaを使えば、それをスキップして、顧客が実際にお金を払ってくれる機能の開発に集中できます。

## 簡単な例

プロジェクト管理SaaS（シンプルなNotionのようなもの）を作っているとしましょう。voltaなしでは、アプリに必要なもの：

- ログインページを作る
- Google/GitHubとのソーシャルログイン連携
- セッション管理システムの構築
- テナント（ワークスペース）システムの構築
- チームメンバーが参加するための招待機能
- ロール管理（誰が管理者で、誰が閲覧者か）
- 会社Aが会社Bのプロジェクトを見られないようにする

voltaがあれば、アプリに必要なのは：

- `X-Volta-Tenant-Id`ヘッダーを読んでどのワークスペースかを知る
- `X-Volta-User-Id`ヘッダーを読んでどのユーザーかを知る
- `X-Volta-Roles`ヘッダーを読んで何ができるかを知る
- プロジェクト管理機能の開発に集中する

あとはvoltaがすべて処理します。
