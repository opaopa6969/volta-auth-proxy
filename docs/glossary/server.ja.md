# サーバー

[English version](server.md)

---

## これは何？

**サーバーは、常に電源が入っていて、常に待機していて、他のコンピュータからの要求に応える準備ができているコンピュータです。注文が来たらいつでも調理できるレストランの厨房のようなものです。**

知らなくても大丈夫です！あなたは毎日、気づかないうちにサーバーとやり取りしています。

---

## 身近なたとえ

レストランを思い浮かべてください：

```
  ┌──────────────────────────────────────────────────┐
  │                                                  │
  │   お客さん            厨房                        │
  │  （ブラウザ）         （サーバー）                  │
  │                                                  │
  │   「ピザを        -->  注文を受ける                │
  │    お願い！」          ピザを作る                  │
  │                       送り返す                    │
  │   「ピザを        <--                             │
  │    どうぞ！」                                     │
  │                                                  │
  │   厨房はいつ料理するか自分で決めません。             │
  │   注文を待って、それに応えます。                    │
  │                                                  │
  └──────────────────────────────────────────────────┘
```

サーバーはまさにこの厨房のように動きます：

1. **待機**する
2. リクエストが来る（誰かがウェブページを見たい、ログインしたいなど）
3. リクエストを**処理**する（ページを探す、パスワードを確認する）
4. レスポンスを**送り返す**（ページ、または「ログイン成功！」）
5. また待機に戻る

---

## サーバーは役割であって、特別なマシンではない

実はサーバーはただの普通のコンピュータです。あなたのノートパソコンもサーバーになれます！「サーバー」という言葉は、コンピュータが何であるかではなく、何を*しているか*（他のコンピュータに奉仕＝serveしている）を表しています。

```
  ノートPCでGoogleを見ている時  = ノートPCはクライアント（要求している側）
  ノートPCでvoltaを動かしている時 = ノートPCはサーバー（要求に応えている側）
```

ただし、プロの環境では、サーバーは通常以下のような特別なコンピュータです：
- 常に電源が入っている（24時間365日）
- 安定したインターネット接続がある
- 安全で温度管理された部屋に置かれている

---

## クライアントとサーバー

何かを要求するコンピュータを**クライアント**と呼びます。応答するコンピュータを**サーバー**と呼びます。お客さんとウェイターの関係のようなものです：

```
  クライアント（ブラウザ） ---「ログインページをちょうだい」--->  サーバー（volta）
  クライアント（ブラウザ） <--「ログインページをどうぞ」------  サーバー（volta）
```

---

## volta-auth-proxyでは

**volta-auth-proxyでは：** volta-auth-proxyはサーバーそのものです。ログインリクエストを待ち、処理し（パスワードの確認、セッションの作成）、ブラウザにレスポンスを送り返します。注文を待って応える厨房と同じです。
