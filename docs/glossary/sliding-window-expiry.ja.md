# スライディングウィンドウ有効期限

[English version](sliding-window-expiry.md)

---

## これは何？

スライディングウィンドウ有効期限とは、ユーザーが操作するたびにセッションの期限が延長されるタイムアウト方式です。図書館で「2時間本に触らなかったら退館してください」と言われるイメージです。本を手に取るたびに、2時間のカウントダウンがリセットされます。

技術的に言うと、セッションは**最初のログイン時刻**からではなく、**最後の操作時刻**から N 時間で期限切れになります。

---

## なぜ重要？

スライディング有効期限がないと、ユーザーは何をしていても固定時刻にログアウトされてしまいます。長い文書を書いている最中に、ずっと作業していたのに「8時間前にログインしたので」と強制ログアウトされたら困りますよね。スライディング有効期限はこの問題を防ぎます。

トレードオフはセキュリティです。延長し続けるセッションは、ユーザー（または攻撃者）が操作し続ける限り理論上は永遠に生き続けます。そのため、多くのシステムではスライディング有効期限と**絶対タイムアウト**（[absolute-timeout.md](absolute-timeout.md) 参照）を組み合わせて上限を設けています。

---

## 簡単な例

```
09:00  ユーザーがログイン       セッション期限 = 17:00（8時間後）
11:30  ページをクリック         セッション期限 = 19:30（今から8時間後）
14:00  ページをクリック         セッション期限 = 22:00（今から8時間後）
14:01  ユーザーが帰宅
22:00  セッション期限切れ。次回はログインが必要。
```

固定タイムアウトだと、操作に関係なく必ず 17:00 に切れてしまいます。

---

## volta-auth-proxy では

volta は **8時間のスライディングウィンドウ**（`SESSION_TTL_SECONDS=28800`）を使っています。

セッション Cookie で認証されるたびに、volta は `touchSession()` を呼び出し、`expires_at` を現在時刻から 8 時間後に更新します：

```java
store.touchSession(session.id(), Instant.now().plusSeconds(config.sessionTtlSeconds()));
```

これにより、アクティブなユーザーは一日中ログインしたまま作業でき、操作をやめてから 8 時間後にセッションが切れます。

volta は現在、絶対タイムアウトの上限は設けていません。ほとんどの SaaS アプリケーションでは、スライディングウィンドウだけで十分なセキュリティと使いやすさのバランスが取れています。
