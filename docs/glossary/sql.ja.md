## これは何？

**SQLはデータベースと会話するための言語です。図書館の司書に本を探してもらったり、追加してもらったり、更新してもらうようなものです。**

知らなくても大丈夫です！SQLはほぼすべてのアプリやウェブサイトの裏側で使われていますが、ほとんどの人は目にすることがありません。

---

## 身近なたとえ

学校の司書が全生徒の名簿を管理していると想像してください：

```
  あなた：「3年A組の生徒を全員見せて」
  司書：  （名簿を調べて、リストをくれる）

  あなた：「9歳の生徒を全員見せて」
  司書：  （名簿をフィルタリングして、短いリストをくれる）

  あなた：「新しい生徒を追加して：さくら、9歳、3年B組」
  司書：  （名簿に新しい行を書き加える）
```

SQLは、コンピュータが理解できるようにこれらのお願いを正式に書く方法にすぎません。

---

## 実際のSQLの例

[データベース](database.ja.md)の記事の学校名簿を使って：

### 「3年A組の生徒を全員見せて」

```sql
SELECT * FROM students WHERE class = '3A';
```

日本語のように読むと：studentsテーブル**から（FROM）**、classが3Aの**ところ（WHERE）**を、全部**取り出して（SELECT）**。

### 「9歳の生徒の名前だけ見せて」

```sql
SELECT name FROM students WHERE age = 9;
```

結果：

```
  name
  ──────────────
  田中ゆき
  鈴木ひろ
  佐藤けん
```

### 「新しい生徒を追加して」

```sql
INSERT INTO students (name, age, favorite_color)
VALUES ('伊藤さくら', 9, '紫');
```

日本語で読むと：studentsテーブル**に挿入（INSERT INTO）**、値は名前が伊藤さくら、年齢が9、好きな色が紫。

### 「めいの年齢を変えて」

```sql
UPDATE students SET age = 10 WHERE name = '渡辺めい';
```

読むと：studentsテーブルを**更新（UPDATE）**、名前が渡辺めいの**ところ（WHERE）**、年齢を10に**設定（SET）**。

## 基本の4つの操作

データに対して行うことは、ほとんどが4つのアクションに集約されます：

```
  ┌──────────────┬──────────────────────────────┐
  │  SQLコマンド   │  何をするか（司書バージョン）  │
  ├──────────────┼──────────────────────────────┤
  │  SELECT       │  「見せて...」                │
  │  INSERT       │  「追加して...」              │
  │  UPDATE       │  「変えて...」                │
  │  DELETE       │  「消して...」                │
  └──────────────┴──────────────────────────────┘
```

これだけです！この4つのコマンドでSQLのほとんどをカバーしています。

---

## なぜ「SQL」と呼ばれるのか？

SQLは「Structured Query Language（構造化問い合わせ言語）」の略です。「Query（クエリ）」は「質問」のかしこまった言い方です。つまりSQLは「構造化された質問の仕方」にすぎません。

発音は「エス・キュー・エル」でも「シークェル」でも、どちらも正しいです！
