## これは何？

**URLはウェブページの住所です。郵便の住所と同じですが、インターネット用です。**

知らなくても大丈夫です！ブラウザのアドレスバーに何かを入力したり、リンクをクリックするたびに、URLを使っています。

---

## 身近なたとえ

郵便の住所を思い浮かべてください：

```
  住所：     〒100-0001 東京都千代田区1-2-3 ABビル4階
  URL：     https://shop.example.com:443/shoes?color=red
```

住所に部分があるように（都道府県、市区町村、番地、建物名）、URLにも部分があります。分解してみましょう。

---

## URLの部分

```
  https://shop.example.com:443/shoes/sneakers?color=red&size=28
  ─────   ────────────────  ─── ─────────────── ───────────────
    │            │            │        │               │
    │            │            │        │               └─ クエリ：フィルター/オプション
    │            │            │        └─ パス：どのページ
    │            │            └─ ポート：どのドア（普段は非表示）
    │            └─ ドメイン：どの建物
    └─ プロトコル：どうやって行くか
```

ショッピングモールで例えてみましょう：

| URLの部分 | モールのたとえ | 例 |
|----------|-------------|---------|
| **プロトコル** (`https`) | 行き方（車、バス） | `https` = 安全、`http` = 安全でない |
| **ドメイン** (`shop.example.com`) | モールの名前 | 「サンシャインショッピングセンター」のようなもの |
| **ポート** (`:443`) | どの入口ドア | 普段は非表示（443はhttpsの既定値） |
| **パス** (`/shoes/sneakers`) | モール内のどの店、どのコーナー | 「靴屋に行って、スニーカーコーナーへ」 |
| **クエリ** (`?color=red&size=28`) | 何を探しているか | 「赤いので28cmがほしい」 |

---

## 「https」の「s」が大切な理由

```
  http://   = はがきを送るようなもの -- 途中で誰でも読める
  https://  = 封筒で送るようなもの -- あなたと相手だけが読める
```

「s」は「secure（安全）」のsです。パスワードや個人情報を入力するときは、必ず`https://`を確認しましょう！

## volta-auth-proxyでは

**volta-auth-proxyでは：** URLはあらゆる場所で使われています。ログインページにはURLがあり、ログイン後のコールバックにもURLがあり、サインイン後にユーザーをどこに送るかを制御するリダイレクトURIもすべて慎重に設定する必要があるURLです。
