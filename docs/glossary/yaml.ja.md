# YAML

[English version](yaml.md)

---

## これは何？

**YAMLは、きれいに整理された買い物リストのような見た目で設定やデータを書く方法です。**

知らなくても大丈夫です！YAMLは情報を整理して書くための形式にすぎません。買い物リストが読めれば、YAMLも読めます。

---

## 身近なたとえ

買い物リストを思い浮かべてください。ごちゃごちゃに書くこともできます：

```
卵、牛乳、パン、りんご3個、チーズ（チェダーとモッツァレラ）
```

でもインデント（字下げ）できれいに書くこともできます：

```
乳製品：
  - 牛乳
  - チーズ：
      - チェダー
      - モッツァレラ

パン売り場：
  - パン

青果：
  - りんご（3個）
  - 卵
```

YAMLは2番目の書き方のようなものです。インデントとダッシュで情報を整理します。コンピュータにも読めて、人間にも目を細めずに読めます。

---

## YAMLはどんな見た目？

人の情報を書いた簡単なYAMLファイルです：

```yaml
name: 田中ゆき
age: 28
hobbies:
  - 読書
  - 料理
  - ハイキング
address:
  city: 東京
  zip: 100-0001
```

ポイント：
- クォーテーションは（通常）不要
- 波括弧や角括弧もなし
- インデント（行頭のスペース）で何が何に属するかを表す
- ダッシュ（`-`）でリストを作る

---

## YAML vs JSON

YAMLとJSONは同じ情報を保存しますが、YAMLの方が読みやすいです：

```
  JSON（記号が多い）：              YAML（すっきり）：

  {                                 name: ゆき
    "name": "ゆき",                 age: 28
    "age": 28,                      hobbies:
    "hobbies": [                      - 読書
      "読書",                         - 料理
      "料理"
    ]
  }
```

人間が読み書きする設定ファイルには、多くのプロジェクトがYAMLを選んでいます。

---

## YAMLで1つだけ注意すること

**スペースが重要です！** YAMLでは、インデントにスペース（タブではなく）を使い、スペースの数が揃っている必要があります。これは正しい：

```yaml
address:
  city: 東京       # スペース2つ -- OK！
  zip: 100-0001   # スペース2つ -- OK！
```

これは壊れています：

```yaml
address:
  city: 東京       # スペース2つ
    zip: 100-0001 # スペース4つ -- あっ！揃ってない！
```

学校の作文のアウトラインと同じです。同じレベルのものは同じインデントにしなければなりません。
